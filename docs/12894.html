Overview

<h1 align="center"><b>NTIRE 2026: 3D内容超分辨率挑战赛</b></h1> <h2>重要时间节点</h2> <ul> <li>2026-01-15: 发布训练和验证数据；<br></li> <li>2026-01-30: 验证服务器上线；<br></li> <li>2026-03-10: 发布最终测试数据，验证服务器关闭；<br></li> <li>2026-03-17: 测试结果提交截止时间；<br></li> <li>2026-03-17: 信息表/代码/模型提交截止时间；<br></li> <li>2026-03-19: 向参赛者发布测试初步成绩；<br></li> <li>2026-03-22: 挑战赛论文提交截止时间；<br></li> <li>2026-03-24: 论文审稿决议通知；<br></li> <li>2026-04-08: 相机就绪版本；<br></li> <li>2026-06-18 (待确认): 研讨会日期；<br><br></li> </ul> <h2>介绍</h2> 除了传统的图像和视频外，3D内容因其能提供场景的外观和几何信息而获得了广泛的欢迎。近年来，由于3D生成和重建技术的进步，3D内容呈现出指数级增长。然而，现有的3D生成模型仍然存在质量相对较低的问题。对于3D重建，由于现实世界中存在的多种约束，观察到的多视图图像通常分辨率较低，这最终会降低重建的3D内容的质量。在AR/VR等应用中，提高这些3D内容的分辨率是高度需要的，以提供沉浸式体验并帮助解析真实世界。<br> 近年来，图像超分辨率(SR)在改进图像视觉质量方面取得了显著进展。为了增强3D内容的质量，一个直接的方法是使用现成的图像SR模型来超分辨渲染的图像。然而，这种方法忽略了几何和外观信息之间紧密耦合的关系，导致多视图不一致和闪烁伪影。<br> 在此挑战赛中，我们旨在建立3D内容超分辨率的基准。我们希望突出3D内容超分辨率所面临的具体挑战和研究问题。我们期望此挑战能激励社区探索低级视觉和3D视觉的交叉领域，最终推动AR/VR等新兴应用中的技术进步。<br><br> <h2>挑战赛描述</h2> 第8届NTIRE: 图像恢复与增强新趋势研讨会将于2026年6月与CVPR 2026一起举办。<br> 本挑战赛的目标是从低分辨率(LR) 3D内容重建高分辨率(HR) 3D内容。在开发阶段，将发布验证集。参赛者可以通过将结果提交到验证服务器来评估其模型在验证集上的性能。具体而言，超分辨率图像在给定的相机内参和外参下渲染，然后提交到服务器进行在线评估。在测试阶段，将发布测试集，其中仅包含LR 3D内容。挑战赛参赛者应将其训练的模型应用于LR 3D内容以生成超分辨率结果。然后，这些结果提交到服务器并由组织者进行评估。<br> 本挑战赛有两个赛道： <ul> <li><a href="https://www.codabench.org/competitions/12891/">赛道1：双立方退化</a><br></li> 该赛道的目标是在双立方退化(Matlab $imresize$双立方模式函数)下获得超分辨率3D内容。 <li><a href="https://www.codabench.org/competitions/12894/">赛道2：真实退化</a><br></li> 该赛道的目标是在真实退化下获得超分辨率3D内容。 </ul> <h2>组织者</h2> <ul> <li><a href="https://longguangwang.github.io/">王龙光</a> (wanglg9@mail.sysu.edu.cn)，中山大学<br></li> <li><a href="http://yulanguo.me/">郭玉兰</a> (guoyulan@sysu.edu.cn)，中山大学<br></li> </ul>

Data

<h1 align="center"><b>NTIRE 2026：3D内容超分辨率挑战赛</b></h1> <h2>赛道1：双三次插值退化</h2> <b>（1）训练集</b><br> 本挑战赛不提供训练集，鼓励参与者使用多种外部数据集，包括但不限于图像SR数据集、深度图SR数据集、多视图数据集和3D重建数据集，以改进3D内容的质量。<br><br> <b>（2）验证集</b><br> 验证集包含LR 3D内容（使用分辨率为342x228的图像训练，<a href="https://www.jianguoyun.com/p/DTTUxjwQ1Zb_DRiz9JoGIAA">第1部分</a>和<a href="https://www.jianguoyun.com/p/DUH4tpMQ1Zb_DRi19JoGIAA">第2部分</a>）、HR图像/深度图（1368x912）及其对应的<a href="https://www.jianguoyun.com/p/DVZDB1gQ1Zb_DRiU9JoGIAA">相机姿态</a>。参与者可下载验证集，通过将其超分辨率图像/深度图与HR图像/深度图进行比较来评估所开发模型的性能。鼓励参与者撰写论文来描述其方法，并使用发布的验证集进行性能评估。<br><br> <b>（3）测试集</b><br> 为了对提交的模型进行排名，提供了包含两个3D场景的测试集。与验证集不同，测试集仅发布<a href="https://www.jianguoyun.com/p/DU4aPaYQ18WLDhjXg54GIAA">LR 3D内容</a>（使用分辨率为342x228的图像训练）和一组<a href="https://www.jianguoyun.com/p/Daju7F0Q1Zb_DRiJ9JoGIAA">相机姿态</a>。参与者必须将其模型应用于发布的LR 3D内容，并将超分辨率图像/深度图（1368x912）在给定相机姿态处提交到服务器。<br><br> <h2>赛道2：真实退化</h2> <b>（1）训练集</b><br> 本挑战赛不提供训练集，鼓励参与者使用多种外部数据集，包括但不限于图像SR数据集、深度图SR数据集、多视图数据集和3D重建数据集，以改进3D内容的质量。<br><br> <b>（2）验证集</b><br> 验证集包含<a href="https://www.jianguoyun.com/p/DSNkhv0Q1Zb_DRih9JoGIAA">LR 3D内容</a>（使用分辨率为342x228的图像训练）、HR图像/深度图（1368x912）及其对应的<a href="https://www.jianguoyun.com/p/DVZDB1gQ1Zb_DRiU9JoGIAA">相机姿态</a>。参与者可下载验证集，通过将其超分辨率图像/深度图与HR图像/深度图进行比较来评估所开发模型的性能。鼓励参与者撰写论文来描述其方法，并使用发布的验证集进行性能评估。<br><br> <b>（3）测试集</b><br> 为了对提交的模型进行排名，提供了包含两个3D场景的测试集。与验证集不同，测试集仅发布<a href="https://www.jianguoyun.com/p/DaVm3REQ18WLDhjfg54GIAA">LR 3D内容</a>（使用分辨率为342x228的图像训练）和一组<a href="https://www.jianguoyun.com/p/Daju7F0Q1Zb_DRiJ9JoGIAA">相机姿态</a>。参与者必须将其模型应用于发布的LR 3D内容，并将超分辨率图像/深度图（1368x912）在给定相机姿态处提交到服务器。<br><br>

Evaluation

<h1 align="center"><b>NTIRE 2026：3D内容超分辨率挑战赛</b></h1> <h2>评估</h2> 本挑战赛的目标是从低分辨率（LR）的3D内容重建高分辨率（HR）的3D内容。在开发和测试阶段，挑战赛参与者需要提交超分辨率处理后的结果。计算评估指标的示例代码可以在<a href="https://drive.google.com/file/d/1cYcP898xBpNWR-bGKKyFECymZkieN11a/view?usp=drive_link">此处</a>找到。<br> <ul> <li><b>赛道1：双三次插值降质</b><br></li> 给定场景的多视图图像，使用双三次插值降质（Matlab中$imresize$函数的双三次模式）生成LR图像。</ul> <ul> <li><b>赛道2：真实降质</b><br></li> 给定场景的多视图图像，采用包含随机模糊、下采样、噪声和压缩的真实降质模型来合成LR图像。 </ul> 对于定量评估，我们为每个测试3D内容选择M个相机位姿来渲染M张图像和对应的深度图。然后，使用PSNR评估视觉质量，而EPE用于评估几何精度。最终排名分数计算方式如下： <div align="center"><img src="https://pic1.imgdb.cn/item/6958eed2da3df73ea1bca10e.png", width="50%"></div> <h2>提交</h2> 在开发阶段，参与者可以在验证集上提交结果以获得Codabench服务器的反馈。在测试阶段，参与者将提交测试集的全部结果。这应该与最后一次向Codabench的提交相匹配。<br> 提交的结果应包含在给定的相机内参和外参下从超分辨率3D内容渲染的图像和深度图（1368x912）。注意，内参和外参文件可以在此处找到（<a href="https://www.jianguoyun.com/p/DVZDB1gQ1Zb_DRiU9JoGIAA">验证位姿</a>和<a href="https://www.jianguoyun.com/p/Daju7F0Q1Zb_DRiJ9JoGIAA">测试位姿</a>），以下示例代码中的相机索引应用于渲染。 <div align="center"><img src="https://i.imgs.ovh/2026/01/19/ybyjfa.png", width="100%"></div> <br> 提交结果时，创建一个ZIP压缩包，包含所有生成的图像和深度图。渲染图像和深度图的命名应与验证集和测试集中的相机内参和外参命名相同。有效的虚拟提交示例文件可以在<a href="https://www.jianguoyun.com/p/DS7pUnsQleb6DRiD4ZoGIAA">此处</a>找到。<br> <div style="background-color: #DDD; border: 5pt"> 测试阶段结束后，参与者需要1）填写<a href="https://docs.google.com/forms/d/e/1FAIpQLSdx7w348lLapHfupaHDwkSRXIhN8Lxzksat1nUIlNtK9v6cnQ/viewform">在线表格</a>，以及2）向官方提交账户（<strong>ntire.3dsr@outlook.com</strong>）通过电子邮件提交一个ZIP压缩包（包含事实表、源代码和最终结果，事实表示例可以在<a href="https://www.jianguoyun.com/p/DbxH8hgQleb6DRi_6pgGIAA">此处</a>找到）。最终提交应遵循以下规则：<br> <ul> <li>提交的结果必须来自生成Codabench最后一次提交的同一方法。我们将检查一致性。否则提交无效。<br></li></ul> <ul> <li>必须提交测试源代码（或可执行文件）和训练代码。可重现性是必要条件。<br></li></ul> <ul> <li>应一起提交描述方法细节的事实表。事实表格式随数据提供。参与者必须提交已编译的PDF文件和TeX源文件。我们鼓励参与者提供详细信息并附带图像源文件。这有助于撰写挑战赛总结报告。<br></li> </ul> </div> 最终测试阶段的每个参与团队应使用提供的事实表模板来描述其解决方案。<br> <h3>电子邮件
