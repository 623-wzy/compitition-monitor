# EBMV 2026 低级图像处理挑战：SEE-600K事件引导亮度调整 ## 概述 真实世界图像常常受到不同光照退化的影响，包括低光、过度曝光、混合照明和高对比度场景。本挑战旨在研究在广泛光照条件下低级图像增强的统一事件引导解决方案。 参赛者需要使用RGB帧和事件数据来恢复清晰、曝光良好且结构保真的图像。本挑战与ECCV 2026事件多模态视觉研讨会相关：成像、感知与理解。 ## 数据集 本挑战基于SEE-600K，出自论文： "SEE: See Everything Every Time - Adaptive Brightness Adjustment for Broad Light Range Images via Events" SEE-600K是在多种光照条件和真实场景下收集的大规模RGB-事件数据集。它提供成对的RGB图像和事件数据用于事件引导亮度调整和低级图像复原。 数据集： https://huggingface.co/datasets/yunfanlu/SEE-600K 基准代码： https://github.com/yunfanLu/SEE 研讨会网站： https://eventbasemultimodalvision.github.io ## 任务 参赛者需要开发一个事件引导的低级图像处理模型。 输入： 1. 在具有挑战性光照条件下捕获的RGB图像。 2. 对应的事件流或事件表示。 3. 可选的元数据或曝光提示（如提供）。 输出： 1. 亮度调整后的RGB图像。 2. 输出应保留场景结构、恢复有用细节，避免不现实的幻觉。 本任务侧重于事件引导的亮度调整和图像增强，而非通用图像生成。 ## 挑战赛道 ### 宽光谱自适应亮度调整 鼓励参赛者在SEE-600K上训练模型。允许使用额外的训练数据，但所有外部数据集、收集设置和预训练模型应在最终技术报告中明确说明。 ## 提交格式 每份提交应上传为压缩ZIP文件。 推荐的结构： submission.zip ├── scene_000001.png ├── scene_000002.png ├── scene_000003.png └── ... 要求： 1. 输出文件名必须与输入样本ID匹配。 2. 图像应保存为PNG文件。 3. 输出图像应为RGB图像。 4. 输出分辨率必须与输入/参考分辨率相同。 5. 不要更改文件名或文件夹结构。 6. 除非明确要求，否则不要包含额外文件。 ## 评估 官方评估将在测试集上进行。 指标： 1. PSNR：像素级重建精度。 2. SSIM：结构相似性。 3. LPIPS：感知相似性。 排名： 1. PSNR是主要排名指标。 2. SSIM和LPIPS将作为次要指标使用。 3. 最终排行榜将基于官方测试集。 验证服务器主要用于开发和提交格式检查。

Overview

# SEE Challenge 2026: 事件引导亮度调整竞赛（SEE-600K数据集） 欢迎参加**SEE Challenge 2026**，由**EBMV @ ECCV 2026**主办。本竞赛研究**广泛光照条件下的事件引导亮度调整**。 目标是从具有挑战性的RGB输入图像和相应的事件相机信号恢复一张曝光良好的RGB图像。参赛者可使用官方基准代码、SEE-600K数据集、外部数据和预训练模型，但必须在最终技术报告中明确标注所有外部资源。 ## 快速开始 | 需求 | 链接 | | --- | --- | | 代码、基准、数据集链接、预训练模型 | [GitHub: yunfanLu/SEE](https://github.com/yunfanLu/SEE) | | 完整的设置、训练、推理和评估教程 | [docs/TUTORIAL.md](https://github.com/yunfanLu/SEE/blob/main/docs/TUTORIAL.md) | | CodaBench打包和上传说明 | [docs/CODABENCH_SUBMISSION_GUIDE.md](https://github.com/yunfanLu/SEE/blob/main/docs/CODABENCH_SUBMISSION_GUIDE.md) | | 中文竞赛指南 | [docs/COMPETITION_GUIDE_ZH.md](https://github.com/yunfanLu/SEE/blob/main/docs/COMPETITION_GUIDE_ZH.md) | | 基准结果 | [docs/TEST_RESULTS.md](https://github.com/yunfanLu/SEE/blob/main/docs/TEST_RESULTS.md) | | 数据集 | [Hugging Face上的SEE-600K](https://huggingface.co/datasets/yunfanlu/SEE-600K) | 推荐工作流程： 1. 克隆仓库并安装依赖。 2. 下载SEE-600K。 3. 使用官方训练集训练或改进模型。 4. 在原始图像尺寸`346 x 260`上进行推理。 5. 使用提供的CodaBench脚本收集预测图像。 6. 将收集的文件夹压缩并上传到CodaBench。 ## 任务 给定： - 在困难照明条件下捕获的RGB图像； - 相应的事件流或事件表示； - 在可用时提供的可选亮度或曝光元数据； 参赛者应输出**亮度调整后的RGB图像**。 输出应： - 保留场景结构和图像细节； - 产生自然的色彩和曝光； - 避免不真实的幻觉内容； - 与提交的原始SEE图像分辨率相匹配。 ## 数据集：SEE-600K 本竞赛采用**SEE-600K**，这是一个用于宽光程范围图像增强的大规模RGB-事件数据集。 | 属性 | 数值 | | --- | --- | | 图像数量 | 610,126 | | 真实场景 | 202 | | 数据模态 | RGB帧 + 事件数据 | | 光照条件 | 低光、正常光、高光、混合光照 | | 图像分辨率 | `346 x 260` | 下载： - Hugging Face: [yunfanlu/SEE-600K](https://huggingface.co/datasets/yunfanlu/SEE-600K) - 其他镜像和说明: [GitHub README](https://github.com/yunfanLu/SEE#dataset-download) 预期的数据集结构和设置说明记录在[docs/TUTORIAL.md](https://github.com/yunfanLu/SEE/blob/main/docs/TUTORIAL.md#2-prepare-the-dataset)中。 ## 训练/测试划分 以下场景组保留用于官方测试，**禁止用于训练**： | 类型 | 保留组 | | --- | --- | | 室内 | 000, 001, 002, 006, 012, 018, 030, 042, 048, 054, 060, 065, 070, 074, 075 | | 室外 | 100, 106, 112, 118, 124, 130, 136, 142, 148, 154, 160, 166, 173, 184, 189, 194, 200, 206, 212, 217, 222, 225 | 所有其他组可用于训练。详见[docs/TUTORIAL.md](https://github.com/yunfanLu/SEE/blob/main/docs/TUTORIAL.md#23-traintest-split)中的划分详情。 ## 基准：SEE-Net 官方仓库提供**SEE-Net**，一个紧凑的基准模型，参数约为**190万**。它使用： - RGB帧； - 事件数据； - 亮度/曝光提示。 基准支持通过提示值`B`进行可控亮度调整： - `B ~= 0.5`：正常曝光； - `B > 0.5`：更亮的输出； - `B < 0.5`：更暗的输出。 基准的设置、训练、推理和评估文档见： - 设置: [docs/TUTORIAL.md#3-set-up-the-python-environment](https://github.com/yunfanLu/SEE/blob/main/docs/TUTORIAL.md)
