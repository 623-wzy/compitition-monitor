Overview

# 基于事件的多模态理解挑战：空间推理赛道 @ECCV 2026 欢迎参加事件理解多模态大语言模型（MLLMs）挑战赛，官方承办于ECCV 2026！ ## 🎯 概述与目标 本挑战赛基准测试MLLMs在基于事件的视觉上的空间推理能力，重点关注物体计数、绝对距离估计和空间关系推理。 ## 📝 任务描述 为全面基准测试模型能力，挑战赛跨3个不同的空间任务评估性能： * **物体计数**：准确枚举场景中存在的特定动态或静态物体的数量。 * **绝对距离**：估计物体之间或从相机到目标物体的物理定量距离（例如，以米为单位）。 * **空间关系**：确定多个物体的相对拓扑位置和结构方向。 ## 📅 重要日期 *所有截止日期均为UTC时间晚上11:59。* * **2026年5月10日**：挑战赛网站开放。 * **2026年5月12日**：训练和验证数据发布。 * **2026年5月20日**：验证服务器上线。 * **2026年6月25日**：测试数据发布与测试服务器上线。 * **2026年7月3日**：测试提交截止与测试服务器关闭。 * **2026年7月10日**：结果公布 *(暂定)*。 * **待定**：技术报告提交。 ## 📊 数据统计（测试集） 该基准涵盖严格标注的测试集，涵盖以下分布： | 任务类别 | 项目数量 | | :--- | :--- | | **绝对距离** | 1202 | | **物体计数** | 414 | | **空间关系** | 301 | ## 🔗 提供的资源 为帮助您快速入门和构建模型，请参考我们的官方资源： * **训练数据**：[通过ModelScope下载](https://modelscope.cn/datasets/XduSyL/EventMind/files) * **学术论文**：[在arXiv上阅读 (2511.18448)](https://arxiv.org/pdf/2511.18448) * **项目主页**：[访问EventBench网站](https://xdusyl.github.io/eventbench.github.io/) ## ⚠️ 挑战赛规则 为维护公平性和科学诚实，所有参赛者必须严格遵守以下规则： 1. **禁止作弊或探测**：严格禁止任何探测测试服务器、操纵提交以提取测试集答案或使用测试数据进行模型训练的行为，违反者将被立即取消参赛资格。 2. **模型规模限制**：为鼓励高效且易获取的模型设计，任何评估模型的总参数数必须 **<= 10B**（100亿参数）。

Data Preparation

# 数据准备指南 要参与EventBench挑战赛，您需要获取训练数据和测试集数据（事件流和问题）。请按照以下步骤准备您的本地环境。 ## 第1步：下载训练集 官方训练数据集**EventMind**托管在ModelScope上。您可以从以下链接下载完整的训练分集： 👉 **[下载EventMind训练集 (ModelScope)](https://modelscope.cn/datasets/XduSyL/EventMind)** ## 第2步：下载测试集（事件流和问题） EventBench挑战赛的测试集包含原始事件流文件（`.npz`）和相应的测试问题（`.json`）。由于文件大小限制，这些文件托管在Hugging Face上。 您必须从官方仓库下载**EventBench**数据集： 👉 **[下载EventBench测试集 (Hugging Face)](https://huggingface.co/datasets/XduSyL/EventBench)** *（注意：CodaBench上`input_data.zip`文件中直接提供的测试问题不包含较大的`.npz`事件文件。您必须从上方Hugging Face链接下载事件流。）* ## 第3步：详细说明和配置 有关数据结构、如何读取`.npz`文件以及如何为您的模型设置数据流水线的详细说明，请参阅官方EventBench项目主页。 👉 **[访问官方EventBench项目主页](https://xdusyl.github.io/eventbench.github.io/)** 项目页面包含全面的文档、代码片段和基线模型示例，帮助您将数据无缝集成到训练和评估工作流程中。

Evaluation Details

# 评估指标和提交指南 ## 提交格式 您的提交必须是一个JSON文件。该文件必须包含一个**平面字典**（键值对），其中键是测试问题`id`，值是您的模型预测的`answer`。 *(注意：请确保您的提交是直接的JSON对象`{...}`，而不是包装在JSON数组`[{...}]`中，以确保与我们的评分引擎兼容。)* **`predictions.json`示例：** ```json { "228350c5-4a8f-4976-abf9-fe52b86bddae": "D", "a959e02d-7ef6-457b-9a16-9c85934ba980": "5.3198", "e495b370-59ba-471b-8642-5ea239380f5a": "4", "d42e3a21-2c72-48ef-9f33-522338dc1c44": "left", "97258721-3d86-4bc2-82d1-481b59e1b7cd": "spatial\ntemporal" } ``` ## 提交方式 1. **准备文件：** 格式化您的模型预测结果，并严格将其保存为`predictions.json`。 2. **压缩：** 直接将`predictions.json`文件压缩成`.zip`存档（例如`submission.zip`）。**不要**在存档中包含任何父文件夹。 3. **上传：** 导航至挑战页面上的**"我的提交"**标签，选择活跃阶段，然后上传您的`.zip`文件。评估服务器将自动处理您的结果。 ## 评估指标 该挑战在3个不同的任务上评估模型。评估服务器根据以下具体匹配规则为每个类别计算准确度： * **绝对距离**：直接距离值输出（例如`5.3198`）。**指标：** 如果预测值与基准值之间的相对误差≤30%，则正确。 * **对象计数**：数值。对象计数的直接输出（例如`4`）。**指标：** 要求完全匹配。 * **空间关系**：字符匹配（例如`left`）。**指标：** 要求完全匹配。 ## 最终评分 评估脚本将独立计算所有3个任务的个别准确度分数。最终排行榜排名严格由**平均分数**决定，即所有任务类别分数的平均值。
