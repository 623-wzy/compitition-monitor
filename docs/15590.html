Overview

# 超声生物测量基础模型（FU_Biometry）- 基本信息 所有时间节点均为2026年，最终结果公布以MICCAI 2026官方截止日期为准。 准确可靠的超声测量对产前、儿科和成人医疗的临床诊断和决策至关重要。当前手工和半自动化方法耗时较长、依赖操作者，且存在高的观察者间和观察者内变异性。自动化和可推广的系统可将测量时间减少50%-80%，并提高临床工作流效率。 尽管以往挑战赛（PSFHS 2023、IUGC 2024/2025、FUGC 2025、FM_UIA 2026）取得进展，但特定任务的深度学习模型在不同扫描仪、协议、解剖结构和临床场景中缺乏鲁棒性和泛化能力。 本挑战赛建立了大规模统一的超声生物测量基准，涵盖产前、分娩期和成人心脏超声，包含超过20个临床参数。旨在推动可推广的多任务、跨域模型的发展，提高测量效率、测量一致性和实际临床应用价值。 # 挑战赛目标 开发自动化、可推广的超声生物测量方法，对设备、医疗中心和人群具有鲁棒性，并通过临床验证进行端到端评估。 # 挑战赛宗旨 1. 提供大规模多中心、多设备数据集，包含专家标注，用于模型训练和验证。 2. 评估解剖学标志点检测和超过20种临床生物测量。 3. 评估从自动测量导出的关键临床生物标志物。 4. 促进AI研究人员与临床影像专家的合作。 5. 推进完全自动超声定量的可靠性，通过标志点误差和临床测量误差进行评估。 # 资源 参赛者可通过以下途径获得基线： https://github.com/lijiake2408/Foundation-Model-for-Ultrasound-Biometry 训练集：发布完整标注，供模型开发使用。 验证集：向参赛者分发原始图像。将预测结果上传至submission.zip提交到平台进行评分。 测试集：由组织者托管。在最终阶段，您的Docker镜像将在该数据上运行。 # 获取数据集方式 参赛者可通过以下途径获取训练数据集： 百度网盘：https://pan.baidu.com/s/14pFgrB0jPxtN3MrMAX4GsQ?pwd=1234 谷歌云盘：https://drive.google.com/drive/folders/1xae7zzKL5YsiIteFJWZEqCfAsSFeGk-z?usp=sharing 如您对任务有任何疑问，请发送邮件至： lijiakekobe@gmail.com ljk.me@qq.com

Evaluation

# FU_Biometry Challenge - 评估方法 ## 1. 评估目标（无优先级顺序） - 优化以下属性的算法性能： - 精确度 - 复杂性 - 硬件要求 - 运行时间 - 鲁棒性 - 灵敏度 - 特异性 ## 2. 核心评估指标 两个指标**等权重50%**，均用于最终排名： ### 2.1 平均径向误差（MRE） - **目的**：测量地标定位精度。 - **计算**：对于具有K个必需地标的病例（预测值{p_k}、参考值{g_k}）： \[MRE =(1 / K) \sum_{k=1}^{K}|| p_k-g_k||_2\] - **单位**：像素；如果有像素间距，转换为毫米（mm）。 - **注意**：仅对每个图像的适用地标子集计算。 ### 2.2 超声参数预测差异 - **目的**：测量临床衍生参数的精度。 - **计算**：平均绝对误差（MAE）|ŷ_j − y_j|（ŷ_j = 预测参数，y_j = 手动测量）以原始物理单位（mm/度）表示。 - **归一化**：使用临床定义的容差范围或四分位间距（IQR）对每个参数进行归一化，防止大范围参数主导评估。 - **报告**：为保证透明性，发布原始误差和归一化误差。 ## 3. 排名计算方法 ### 3.1 分数汇总 - 最终得分 = **50% 归一化MRE + 50% 归一化参数误差**（等权重以平衡几何精度和临床测量可靠性）。 ### 3.2 排名框架（ChallengeR风格的排名-后聚合） 1. 跨4个预定义数据域的域平衡加权（避免单一采集源的主导）。 2. 双排行榜：**总体排名** + **域分层排名**（保证透明性）。 3. 并列打破规则：优先考虑测量精度更高（归一化参数误差更低）（临床相关性更高）。 ### 3.3 缺失结果处理 - 缺少测试用例结果的提交通过为相应指标分配所有有效提交中的**最差性能分数**进行处罚。 ## 4. 统计分析 ### 4.1 软件工具 - R语言；Python库（SciPy、Pandas）（在科学研究中被广泛接受）。 ### 4.2 缺失数据处理 - 无多重填补；缺失输出被罚（最终结果无缺失值）。 - 报告所有指标的95%引导置信区间（CI），1,000次重抽样。 ### 4.3 性能精度评估 - **百分位数引导法95% CI（1,000次重抽样）**用于测试集上所有算法性能估计。 ### 4.4 性能变异性评估 - 跨测试用例：通过标准差（SD）、四分位间距（IQR）、箱线图/小提琴图和明确异常值报告的逐案例指标分布。 - 排名变异性：使用**引导法**（非参数，最少假设）评估。 ### 4.5 统计显著性检验 - **成对Wilcoxon符号秩检验**用于算法性能的成对比较（非参数，适用于非正态分布指标）。 - 多重比较校正（Holm或Benjamini-Hochberg）以避免假阳性。 - 效应量（如Cliff's delta）与p值一同报告，便于临床解释。 - 域/设备分层分析以阐明跨域泛化。 ### 4.6 数据假设验证 - 所有统计方法的预验证测试： 1. Shapiro-Wilk检验：评估数据正态性。 2. Levene检验：评估方差齐性。 ## 5. 进一步分析 1. 由排名最高的团队算法构建的**集成模型**的性能验证。 2. 算法间变异性、提交方法的常见偏差/问题的分析。 3. 跨不同数据域/设备排名变异性的深入分析。

Participation Rules

# FU_Biometry Challenge - 参与规则 ## 1. 竞赛生命周期 - 重复性赛事，具有固定的年度会议投稿截止期 ## 2. 参与政策 ### 2.1 算法开发 - 参赛者必须使用所有无标签数据来开发半监督方法。 - 仅允许使用公开可用的外部数据集和预训练模型进行训练；禁止对公开数据进行额外的私人标注。 ### 2.2 组织机构参赛者 - 可以参与竞赛，但**无获奖资格**且**不在排行榜中列出**。 ## 3. 提交规则 ### 3.1 提交方法 - 测试阶段：每支队伍仅允许提交**一个成功的Docker容器**（防止在隐藏测试集上重复调优）。 - 公开验证排行榜与私有最终测试评估严格分离；最终排名仅基于未见过的测试集。 - 排名靠前的提交需要进行人工验证（Docker可执行性、方法描述、规则符合性）。 ### 3.2 验证阶段提交 - 队伍在总共**10次**提交中，每天最多**2次**提交。 - 验证分数仅用于完整性和格式的合理性检查，不用于排名或比较评估。 ## 4. 奖励政策 1. 为排名靠前的队伍、最具创新方法的队伍以及方法描述最详细的队伍提供证书和现金奖励。 2. **前10支队伍**受邀在MICCAI 2026进行口头报告，并鼓励向《IEEE生物医学工程学报》(TBME) 特刊投稿扩展论文。 3. 竞赛综述论文的共同作者身份，该论文将投稿至顶级期刊（MedIA/TMI）。 ## 5. 结果公布 - 组织方公布**排名前10的最优方法**。 - 公布内容包括队伍名称及基于官方评估指标的总体排名。 ## 6. 发表政策 ### 6.1 作者资格 - 前10支队伍的成员，若在研究的概念化、设计、执行或解释中做出重大贡献，符合作者资格。 ### 6.2 独立发表 - 参赛队伍可以独立发表结果，同时尊重竞赛的集体努力。 ### 6.3 禁发期 - 组织方首先发表全面的竞赛论文；建议各队伍在此期间避免独立发表完整研究成果。 ## 7. 代码可用性要求 - 算法代码发布（通过GitHub仓库）和相应的会议论文投稿是**获奖资格**和进一步数据使用的**先决条件**。
