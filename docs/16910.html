# 如何参赛 ArGuard 2026 — 任务1 / 赛道A · **子任务A2: 细粒度类别预测（多标签）**。 ## 参赛资格 您可以以个人或团队身份参赛。一个人可以参与ArGuard 2026中的多个共享任务 — 请在[任务网站](https://araieval.gitlab.io/ArGuard2026/)查看完整列表。 ## 如何报名 1. 在此CodaBench竞赛页面注册。 2. 阅读[条款](#)选项卡 — 特别注意提交文件格式和命名规范。 3. 从Hugging Face Hub下载数据：[`QCRI/ArGuard-Task1`](https://huggingface.co/datasets/QCRI/ArGuard-Task1)。 4. 克隆启动工具包：[`araieval/ArGuard-2026-tasks`](https://github.com/araieval/ArGuard-2026-tasks)。 5. 训练多标签模型，在开发阶段对`dev_test`分割集生成细粒度预测，在最终评估阶段对`test`分割集生成预测。 ## 提交 - 每个阶段由一名团队成员通过**提交**选项卡提交。 - 将JSONL预测文件作为`prediction.zip`上传（见条款）。 - 截止日期前的最新有效提交将作为该阶段您的团队最终提交。 ## 系统描述论文 鼓励所有团队为工作坊提交系统描述论文。如果您参与了多个子任务，一篇论文可以涵盖多个子任务。 ## 联系方式 有问题请联系：arguard2026-organizers@googlegroups.com

Dataset

# 数据集 ArGuard Task 1 — Track A 基于阿拉伯语多模态表情包数据集构建：每个实例包含一张带有叠加阿拉伯文本的图像（通过OCR提取）。表情包使用二元标签和一个或多个**细粒度**子类型进行人工标注。 对于**子任务A2**，每个表情包从**统一的10类活跃词汇**中标记零个或多个标签： - 仇恨子类型（7个活跃）：`轻蔑`、`非人化`、`排斥`、`煽动`、`贬低`、`嘲弄`、`辱骂` - 非仇恨子类型（2个）：`幽默`、`讽刺` - 共享（1个）：`其他` 完整注解分类法还包括5个零支持仇恨类别——`极端主义`、`历史性`、`侮辱`、`刻板印象`、`威胁`——这些类别被格式检查器接受，但**不计分**（在发布的数据中无正例支持）。 ## 获取数据 官方数据分割位于Hugging Face Hub的[`QCRI/ArGuard-Task1`](https://huggingface.co/datasets/QCRI/ArGuard-Task1)。使用启动工具包中的助手脚本下载： ```bash git clone https://github.com/araieval/ArGuard-2026-tasks.git cd ArGuard-2026-tasks/task1 pip install -r requirements.txt python data/download_data.py ``` ## 数据分割 | 分割 | 记录数 | 标签 | 阶段 | 目的 | |--------------|------:|----------|----------------|--------------------------------| | `train` | 3,500 | 完整 | 开发阶段 | 模型训练 | | `dev` | 500 | 完整 | 开发阶段 | 超参数选择、本地评估 | | `dev_test` | 500 | **无** | 开发阶段（排行榜）| 开发阶段排行榜的提交目标 | | `test` | 500 | 完整 | 最终评估 | 盲测排名（三重标注黄金标准） | `dev_test`分割的标签已移除——这是您在开发阶段向排行榜提交预测的文件。 ## 记录模式（`download_data.py`后） ```json { "id": "f9a8…b1.jpg", "image_path": "img/f9a8…b1.jpg", "text": "OCR提取的阿拉伯文本…", "label": "仇恨" | "非仇恨" | null, "fine_grained_label": ["嘲弄", "煽动"], "annotations": [] } ``` ## 许可证 该数据集以**CC BY-NC 4.0**许可证发布（仅供非商业研究使用）。 ## 资源 - 启动工具包：<https://github.com/araieval/ArGuard-2026-tasks> - 任务网站：<https://araieval.gitlab.io/ArGuard2026/>
