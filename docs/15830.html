Overview

# MEG 脑编码挑战 ## 目标 构建一个模型，预测人脑在自然自由浏览视觉场景时的响应方式。具体来说，你需要预测**固视对齐的 MEG 活动** — 即每次眼睛固视开始后 110 毫秒时在 204 个 MEG 梯度计通道上测量的神经反应模式。 ## 背景 数据来自**主动视觉语义学（AVS）**数据集，其中参与者在自由探索自然场景照片的同时，他们的眼动和全头 MEG 被同时记录。 你获得来自五个训练被试（被试 1–5）的数据，并需要泛化到**保留的被试（被试 60）**，其场景浏览行为和相关 MEG 反应构成评估集。 ## 任务 **挑战 1 — 110 毫秒的快照编码** 根据固视元数据（场景 ID、凝视坐标、持续时间、序列位置等）和相应的场景图像，预测固视开始后 110 毫秒处 204 个 MEG 梯度计的幅度向量。 **提交形状：** `(7750, 204)` — 开发集中每个固视一行，204 个通道。 ## 时间表 | 日期 | 事件 | |------|------| | 2026 年 4 月 | 挑战开启 | | 2026 年 7 月 | 挑战关闭 | ## 限制 - 每天 5 次提交 - 总共 50 次提交

Data

# 数据 ## 你将获得的内容 所有数据通过课程仓库分发。你**不需要**将数据上传到 Codabench — 只需上传你的预测结果。 ### 训练数据（被试 1–5） | 文件 | 描述 | |------|-------------| | `training/meg_110ms.npy` | 110 ms 处的脑磁图，形状 `(n_fixations, 204)` | | `training/metadata.csv` | 每个凝视点的元数据（见下表列名） | | `training/channel_names.txt` | 204 个梯度计通道名称的有序列表 | | `avs_scenes/` | 完整场景图像（被试间共享） | ### 开发集（被试 60） | 文件 | 描述 | |------|-------------| | `subject60/challenge1_dev/metadata.csv` | 7750 个凝视点，需要提交预测 | 被试 60 的真实脑磁图被**隐藏**，仅用于评分。 ## 元数据列 `metadata.csv` 中的关键列： | 列名 | 描述 | |--------|-------------| | `sceneID` | 整数场景标识符 — 链接至图像文件 | | `mean_gx`, `mean_gy` | 屏幕中心坐标系中的平均凝视位置（原点 = 中心，y 轴向上） | | `duration` | 凝视持续时间 [毫秒] | | `fix_sequence` | 试验内凝视序号（1 = 首次凝视） | | `time_in_trial` | 试验开始后的时间 [毫秒] | | `subject` | 被试标识符 | ## 场景图像 场景图像为 JPEG 文件，命名格式 `{sceneID:012d}_MEG_size.jpg`，缩放至呈现分辨率（1024 × 768 像素）。凝视块（112 × 112 像素，以凝视位置为中心）可使用 `megcourselib.data.stim.extract_crop()` 提取。

Evaluation

# 评估 ## 指标 提交内容使用**皮尔逊相关系数(r)**进行评估，该系数在**204个通道中各自独立计算**，涵盖开发集中所有7750个注视点。 这产生一个向量 **r** ∈ ℝ^204，据此推导四个排行榜分数： | 列 | 定义 | |--------|-----------| | **平均 r** *(主要)* | `mean(r)` — 所有通道的平均编码 | | **中位数 r** | `median(r)` — 稳健的集中趋势 | | **最大 r** | `max(r)` — 最佳单通道编码 | | **最差10通道** | `mean(r[argsort(r)[:10]])` — 最差10个通道的平均值 | 排名由**平均 r**决定（越高越好）。所有其他列仅用于诊断目的。 ## 提交格式 1. 创建一个形状为 `(7750, 204)`、数据类型为 `float64` 的NumPy数组。 - 行顺序必须与 `subject60/challenge1_dev/metadata.csv` 完全匹配。 - 列顺序必须与 `channel_names.txt` 完全匹配（MEG0001 … MEG0204）。 2. 将其保存为 `predictions.npy`： ```python import numpy as np np.save("predictions.npy", pred) # 形状 (7750, 204) ``` 3. 压缩它（文件必须在zip的根目录，不在子文件夹中）： ```bash zip submission.zip predictions.npy ``` 4. 通过提交按钮上传 `submission.zip`。 ## 有效性检查 评分程序拒绝以下提交： - 形状错误（非7750 × 204） - 包含NaN或Inf值 无效提交在所有列上的得分为0.0。
