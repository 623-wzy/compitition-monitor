## 玩具基准 v2：成对点跟踪 该基准评估了图像数据上的简单**成对点跟踪**任务。 ### 任务定义 对于每个测试样本，参与者获得： - 参考图像 (`ref`) - 目标图像 (`tgt`) - 在参考图像上定义的参考点 `(x_ref, y_ref)` 目标是**预测目标图像上相应的点位置 `(x, y)`**。 ### 评估 使用**预测点与目标图像上真实点之间的欧几里得距离**来评估预测。 最终得分是**所有测试样本上的平均误差**（越低越好）。

## 玩具基准 v2（基于摄入的） 该基准使用**基于摄入的评估协议**来评估**点对跟踪**任务的算法。 参与者需要提交可执行的算法，而不是预先计算的预测结果。提交的算法在受控环境中执行以生成预测，然后自动进行评估。

Data

## 数据描述 本基准测试为提交的算法提供数据作为输入。 ### 输入数据结构 每个阶段提供以下数据作为输入： - **训练数据**：可被算法用于训练或自适应 - **测试数据**：用于在评估过程中生成预测 真实标注**未包含**在输入数据中，为了评估目的而保密。 ### 注意事项 - 所有数据均以预定义的目录结构提供 - 算法必须从提供的输入目录读取数据 - 算法不得依赖外部数据源

Submission

## 提交说明 参赛者需要提交**算法实现**，而不是预测文件。 每次提交必须包含可执行代码，该代码可由基准测试摄入程序运行 以生成所提供测试数据的预测结果。 --- ## 提交内容 有效的提交必须包括： - 实现追踪算法的源代码 - 与摄入接口兼容的模型定义 - 在提供的环境中执行所需的所有文件 提交必须遵循摄入程序期望的目录和接口约定。 --- ## 执行协议 对于每次提交： 1. 摄入程序加载提交的算法 2. 训练数据作为输入被提供 3. 执行算法以在测试数据上生成预测 4. 预测传递到评分程序进行评估 --- ## 限制条件 - 提交不得执行网络访问 - 提交必须在分配的资源限制内运行 - 执行期间仅可访问提交包中包含的文件 --- ### 必需接口 文件 `model.py` 必须定义一个名为 `Model` 的类，包含以下方法： ``` class Model: def fit(self, train_data): """ train_data: 从 train.csv 加载的 numpy 数组或结构化数据 """ pass def predict(self, test_data): """ test_data: 从 test.csv 加载的 numpy 数组或结构化数据 返回： predictions: 形状为 (N, 2) 的 numpy 数组 每一行对应 (x_pred, y_pred) """ return predictions ```

Evaluation

## 评估协议 提交算法生成的预测结果使用**欧几里得距离**来评估预测点与真值点之间的差异。 ### 评分 对于每个测试样本，误差计算公式如下： \[ \sqrt{(x_{pred} - x_{gt})^2 + (y_{pred} - y_{gt})^2} \] 最终得分为**所有测试样本的平均误差**。 得分越低表示性能越好。 --- ## 注意事项 - 参赛者无法获取真值标注 - 评估过程完全自动化 - 所有阶段使用相同的评估协议
