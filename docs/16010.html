Overview

# 赛道 3：Seephys Pro - 视觉物理推理 <div style="padding:18px 22px;border:1px solid #d8e2ef;border-radius:10px;background:#f7fbff;margin:12px 0 22px 0;"> <h2 style="margin:0 0 8px 0;">涵盖图表、表达式和解释的多模态物理推理挑战</h2> <p style="margin:0;font-size:15px;line-height:1.55;"> Seephys Pro 评估 AI 系统是否能够解决需要视觉感知、数学推理和忠实解释的物理问题。该基准结合了结构化物理问题、图表接地的问题和精选的仅图像扩展。为了严格评估系统在未见环境中的泛化能力，该基准纳入了通过摄影捕获的专门分布外（OOD）测试集，由第 5 级数据集表示。第 5 级数据集由传音（Transsion）提供，这是一家专注于新兴市场的领先智能设备和移动互联网公司。它由传音基于 AI 照片的问题解决服务中的匿名服务数据组成。该级别在现实教育场景中评估模型，包括多种问题格式和自然图像质量变化。参与者提交最终答案和推理过程；提交内容由基于 OpenAI 的判断器与参考答案进行对比评分。 </p> </div> 物理推理是一项核心科学能力。它要求系统识别物理量、解读图表、将视觉证据与方程相关联，并得出可防守的结论。最近的多模态模型取得了快速进展，但含有图表和形式表达式的物理问题仍然暴露了"看、读、推理"之间的困难鸿沟。 Seephys Pro 的核心问题是： > **我们如何构建能够可靠整合视觉信息、物理定律和数学推导以解决真实物理问题的 AI 系统？** 该挑战被设计为针对该问题的简洁但高要求的公开基准。 ## 挑战概览 | 项目 | 详情 | |---|---:| | 挑战名称 | Seephys Pro | | 公开排行榜题目数 | 830 | | 私有排行榜题目数 | 3320 | | 必需提交字段 | `question_id`、`prediction`、`reasoning` | | 评分方式 | OpenAI API 语义判断器 | | 总体评分 | 题目数加权准确率 | ## 数据集 数据集托管在 Hugging Face： https://huggingface.co/datasets/Kun-Xiang/SeePhysPro ## 提交内容 参与者必须提交包含以下两项的 `.zip` 文件： - `prediction.csv` - `prediction_private.csv` 每个文件必须包含 `question_id`、`prediction` 和 `reasoning`。 `prediction.csv` 用于 Codabench 公开排行榜。`prediction_private.csv` 用于截止日期后的离线私有排行榜。没有 `prediction_private.csv` 的提交不符合获奖资格。 ## 说明 公开排行榜旨在实现快速迭代。私有评估不由此 Codabench 包自动运行。 ## 引用 <pre style="background-color: #f6f8fa; padding: 16px; border-radius: 6px; line-height: 1.45;"> @article{xiang2026seephyspro, title={SeePhys Pro: Diagnosing Modality Transfer and Blind-Training Effects in Multimodal RLVR for Physics Reasoning}, author={Xiang, Kun and Zhang, Terry Jingchen and Liu, Zirong and Zhou, Bokai and Tang, Yueling and Yu, Junjie and Lu, Jiacong and Huang, Shangrui and Li, Heng and Zhang, Likui and Liu, Kunkun and Zhang, Changzheng and Fang, Yangle and Guo, Boqiang and Zhen, Hui-Ling and Tu, Dandan and Huang, Yinya and Liang, Xiaodan}, journal={arXiv preprint arXiv:2605.09266}, year={2026} } </pre>

# SeePhys Pro 评估 <div style="padding:16px 20px;border:1px solid #eadfb8;border-radius:8px;background:#fffaf0;margin:12px 0 20px 0;"> <strong>重要提示：</strong>每次提交必须同时提供最终答案和支持性解释。仅包含最终答案而没有推理过程的回答将不被视为完整提交。 </div> ## 提交格式 请提交一个包含公开和私密预测文件的 `.zip` 文件，文件位于压缩包根目录。 您的提交应包含： ```text submission.zip ├── prediction.csv └── prediction_private.csv ``` ### 必需文件 #### 1. `prediction.csv` 此文件用于 Codabench 公开排行榜。 #### 2. `prediction_private.csv` 此文件保留给私密排行榜使用。 私密排行榜**不会**在 Codabench 上自动评估。竞赛截止期限后，组织者将下载最终提交，使用私密参考答案离线评估 `prediction_private.csv`。 ### 必需字段 `prediction.csv` 和 `prediction_private.csv` 都必须包含以下字段： ```text question_id prediction reasoning ``` ### 重要说明 Codabench 上显示的公开排行榜分数仅根据 `prediction.csv` 计算。 私密排行榜分数仅根据 `prediction_private.csv` 离线计算。 如果缺少 `prediction_private.csv`，提交将无法获得有效的私密排行榜分数。 ## CSV 示例 ```csv question_id,prediction,reasoning level1_testmini_000000,BD,"两个电路分支给出了所述的加速度和电荷关系，所以 B 和 D 正确。" level1_testmini_000001,1.44,"使用斯涅尔定律和测量的几何关系，折射率约为 1.44。" ``` ## JSONL 示例 ```jsonl {"question_id":"level1_testmini_000000","prediction":"BD","reasoning":"感应电流关系使 B 和 D 正确。"} {"question_id":"level5_testmini_000000","prediction":"A","reasoning":"比较标记的物理量后，图表与选项 A 相匹配。"} ``` ## 评估规则 提交将根据隐藏的参考答案进行评估。对于每个问题，评估系统检查以下内容： 1. 是否提供了最终答案。 2. 是否提供了推理字段。 3. 推理是否与问题相关。 4. 提交的最终答案是否等同于参考答案。 5. 推理是否支持提交的最终答案。 6. 如适用，是否遵守了必需的单位。 如果回答包含以下任何问题，可能无法获得分数： - 缺少最终答案； - 缺少推理； - 提交格式格式不正确或无法读取； - 无关或空白的推理； - 与提交答案相矛盾的推理； - 最终答案不正确； - 当需要单位时缺少或单位不正确。 ## 排行榜指标 公开排行榜报告在已发布的公开评估集上的性能。 | 指标 | 问题数 | 总体权重 | |---|---:|---:| | 第 1 级 | 200 | 200 / 830 | | 第 2 级 | 200 | 200 / 830 | | 第 3 级 | 200 | 200 / 830 | | 第 4 级 | 200 | 200 / 830 | | Solve150 Testmini | 30 | 30 / 830 | 总体分数计算方式为： ```text overall = total_correct_questions / 830 ``` 排行榜还报告： - `Judged`：成功评估的行数； - `Missing`：缺少预测或推理的行数； - `Invalid`：格式不正确或缺少必需字段的行数。 ## 数据集发布 Hugging Face 数据集以公开无答案版本发布供参与者使用。它意图排除以下真实标签字段： - `answer` - `unit` - `error_range` 这些字段保留用于私密评估，不向参与者分发。 ## 说明 参与者负责确保所有 `question_id` 值与已发布数据集完全匹配。同一 `question_id` 的重复预测不被允许，可能被视为无效。
