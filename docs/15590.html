Overview

# 超声生物测量基础模型 (FU_Biometry) - 基本信息 所有时间节点均为2026年，最终结果公布时间以MICCAI 2026官方截止日期为准。 准确可靠的超声测量对于产前、儿科和成人临床诊断和决策至关重要。目前的手动和半自动方法耗时长、依赖操作者，且观察者间和观察者内变异性高。自动化和通用系统可将测量时间减少50%-80%，提高临床工作流效率。 尽管前期挑战赛取得进展（PSFHS 2023、IUGC 2024/2025、FUGC 2025、FM_UIA 2026），任务特定的深度学习模型在不同扫描仪、协议、解剖结构和临床设置中缺乏鲁棒性和泛化能力。 本挑战赛建立了大规模统一的超声生物测量基准，涵盖产前、分娩期和成人心脏超声，包含超过20个临床参数。旨在推动具有更好效率、测量一致性和实际临床应用价值的可泛化、多任务、跨域模型的发展。 # 挑战赛目标 开发自动化、可泛化的超声生物测量方法，对设备、医疗中心和人群具有鲁棒性，并进行临床验证的端到端评估。 # 挑战赛宗旨 1. 提供具有专家标注的大规模多中心、多设备数据集，用于模型开发和验证。 2. 评估解剖学地标检测和超过20项临床生物测量。 3. 评估由自动测量衍生的关键临床生物标志物。 4. 促进AI研究人员与临床影像专家的合作。 5. 推进可靠的全自动超声量化，通过地标误差和临床测量误差进行评估。 # 资源 参赛者可通过以下渠道获取基线： https://github.com/lijiake2408/Foundation-Model-for-Ultrasound-Biometry 训练集：发布完整标注，供模型开发使用。 验证集：向参赛者分发原始图像。上传预测结果至submission.zip到平台进行评分。 测试集：由组织者托管。最终阶段将在此数据上运行您的Docker镜像。 # 获取数据集方式 参赛者可通过以下渠道获取训练数据集： 百度云盘：https://pan.baidu.com/s/14pFgrB0jPxtN3MrMAX4GsQ?pwd=1234 谷歌云盘：https://drive.google.com/drive/folders/1xae7zzKL5YsiIteFJWZEqCfAsSFeGk-z?usp=sharing 如有任何关于任务的问题，请发送邮件至： lijiakekobe@gmail.com ljk.me@qq.com

# FU_Biometry Challenge - 评估方法 ## 1. 评估目标（无优先级顺序） - 在以下属性中优化算法性能： - 准确性 - 复杂性 - 硬件需求 - 运行时间 - 鲁棒性 - 灵敏度 - 特异性 ## 2. 核心评估指标 两个指标具有**相等的50%权重**，均用于最终排名： ### 2.1 平均径向误差（MRE） - **目的**：测量地标定位准确性。 - **计算**：对于具有K个所需地标的情况（预测值{p_k}，参考值{g_k}）： \[MRE =(1 / K) \sum_{k=1}^{K}|| p_k-g_k||_2\] - **单位**：像素；如果有像素间距可用，转换为毫米（mm）。 - **注意**：仅针对每张图像的适用地标子集计算。 ### 2.2 超声参数预测差异 - **目的**：测量临床衍生参数的准确性。 - **计算**：平均绝对误差（MAE）|ŷ_j − y_j|（ŷ_j = 预测参数，y_j = 手动测量），以原始物理单位（mm/度）表示。 - **归一化**：使用临床定义的容差范围或四分位数间距（IQR）对每个参数进行归一化，以防止大范围参数主导评估。 - **报告**：为透明起见，发布原始误差和归一化误差。 ## 3. 排名计算方法 ### 3.1 分数汇总 - 最终分数 = **50%归一化MRE + 50%归一化参数误差**（等权重以平衡几何精度和临床测量可靠性）。 ### 3.2 排名框架（ChallengeR风格的排名后聚合） 1. 在4个预定义数据域中进行域平衡加权（避免单一采集源的主导地位）。 2. 双排行榜：**整体排名** + **域分层排名**（为了透明性）。 3. 平局破除规则：测量准确性更高（归一化参数误差更低）优先（临床相关性更高）。 ### 3.3 缺失结果处理 - 缺少测试用例结果的提交通过为相应指标分配所有有效提交中的**最差性能分数**而受到处罚。 ## 4. 统计分析 ### 4.1 软件工具 - R语言；Python库（SciPy、Pandas）（在科学研究中被广泛接受）。 ### 4.2 缺失数据处理 - 不进行多重填补；缺失输出受到处罚（最终结果无缺失值）。 - 报告所有指标的自助法95%置信区间（CI），采用1,000次重新取样。 ### 4.3 性能精度评估 - **百分位自助法95%置信区间（1,000次重新取样）**用于测试集上的所有算法性能估计。 ### 4.4 性能变异性评估 - 跨测试用例：通过标准差（SD）、四分位数间距（IQR）、箱线图/小提琴图和显式异常值报告，报告逐案例指标分布。 - 排名变异性：使用**自助法**评估（非参数，假设最少）。 ### 4.5 统计显著性检验 - **配对Wilcoxon符号秩检验**用于算法性能的成对比较（非参数，适应非正态指标分布）。 - 多重比较修正（Holm或Benjamini-Hochberg）以避免假阳性。 - 效应量（例如Cliff's delta）与p值一起报告，用于临床解释。 - 域/设备分层分析以背景化跨域泛化。 ### 4.6 数据假设验证 - 所有统计方法的预验证测试： 1. Shapiro-Wilk检验：评估数据正态性。 2. Levene检验：评估方差齐性。 ## 5. 进一步分析 1. 由排名前列团队算法构建的**集成模型**的性能验证。 2. 算法间变异性、提交方法的常见偏差/问题分析。 3. 不同数据域/设备间排名变异性的深入分析。
