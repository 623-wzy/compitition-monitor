Core Task

# 核心任务 MVAA挑战赛旨在反映多个成像模态下的真实临床工作流程。该挑战赛结构包含三个主要分割任务。 ### 参赛和模型提交要求 参赛者需要完成并提交全部三个任务的结果。官方最终排名将根据**最终排名**页面中描述的归一化最终得分确定；因此，仅提交一个或两个任务的结果将不足以获得最终排名。 但是，参赛者不需要使用单一的统一多模态模型。每个任务可以独立处理，参赛者可以为CT、3D TEE和手术视频帧分割任务分别训练和提交不同的模型。 ### 任务1：心脏CT中的二尖瓣分割 该任务重点关注从三维心脏CT扫描中进行二尖瓣多类解剖结构的自动分割。 * **目标：** 参赛者需要分割二尖瓣环以及前/后瓣叶。 * **临床意义：** 高分辨率CT分割提供了精确的空间几何信息，用于术前规划，包括器械尺寸确定和修复策略选择，适用于TEER或瓣环成形术等介入治疗。 ### 任务2：三维经食管超声心动图(3D TEE)中的二尖瓣分割 该任务涉及从容积3D TEE(超声)数据中进行瓣膜结构分割，其作为瓣膜功能的关键术中参考。 * **目标：** 从3D超声容积数据中描绘瓣叶轮廓。 * **技术重点：** 算法必须足够稳健以处理超声特有的伪影、声学噪声、信号丢失以及瓣叶与周围心脏组织间的模糊边界。 ### 任务3：手术视频帧中的二尖瓣分割 该任务重点关注从术中手术视频中提取的单个帧中提取解剖学信息。 * **目标：** 从外科医生直接视角看到的二尖瓣结构进行像素级分割。 * **方法学：** 虽然数据源自视频记录，但该任务被定义为基于帧的分割问题。参赛者应将提供的帧作为单个2D图像进行训练和推理。 * **临床重点：** 该任务能够在实际介入治疗中验证瓣叶运动和结构连续性，在手术环境中提供实时反馈。

# 评估 ## 最终测试评估模式 在最终隐藏测试阶段，参与者通过CodaBench提交Docker镜像描述符。组织方CodaBench工作线程将接受的描述符转发给MVAA GPU运行器，该运行器拉取每个提交的镜像，在隐藏测试集上运行推理，计算官方指标，并将结果得分返回给CodaBench排行榜。 指标在组织方服务器上的公共CodaBench环境外进行计算。此门户中排行榜列显示由该运行器返回的官方隐藏测试原始指标。官方总体排名基于**最终排名**页面中描述的标准化指标得分。 ## 指标 所有任务使用骰子相似系数（DSC）、豪斯多夫距离（HD）和平均表面距离（ASD）进行评估。DSC越高越好；HD和ASD越低越好。 ### 骰子相似系数（DSC） DSC测量预测分割与真实标注之间的重叠程度。计算方式为交集的两倍除以预测和参考前景大小之和。 ### 豪斯多夫距离（HD） HD测量预测和参考掩模之间最大的双向表面偏差。表面从二值掩模中提取，当有间距元数据时使用图像间距计算欧几里得距离。 ### 平均表面距离（ASD） ASD是预测和参考掩模之间对称的表面间平均距离。 ## 任务特定计算 ### 任务1：CT分割 - 前景标签：`1` - 按病例指标：二值前景掩模上的DSC、HD、ASD - 最终任务得分：所有任务1病例的均值 ### 任务2：TEE分割 - 前景标签：`1`和`2` - 为每个前景类分别计算指标，在每个病例内进行平均 - 最终任务得分：所有任务2病例的均值 ### 任务3：视频/图像分割 - 预测前景：任何正像素值 - 参考前景：病例特定的`target_label` - 最终任务得分：所有任务3帧/病例的均值 ## 评估说明 - 当可用时，HD和ASD使用物理间距；否则使用单位间距。 - 缺失或无效预测收到惩罚指标：`DSC = 0`、`HD = 大惩罚值`、`ASD = 大惩罚值`。 - 如果评估的某个类的预测和参考都为空，该类被视为完美匹配：`DSC = 1`、`HD = 0`、`ASD = 0`。 - JSON文件中的预测路径必须是对应任务输出文件夹下的相对路径。绝对路径和`../`遍历会被拒绝。 ## 排行榜列 最终排行榜报告： - `Task1_DSC`、`Task1_HD`、`Task1_ASD` - `Task2_DSC`、`Task2_HD`、`Task2_ASD` - `Task3_DSC`、`Task3_HD`、`Task3_ASD` - `Evaluation_Status` - `Runtime_Seconds` `Evaluation_Status = 1`表示组织方运行器成功完成。`Evaluation_Status = 0`表示描述符验证、镜像拉取、推理或评分失败；失败的提交将收到惩罚指标。 ## 参与者结果摘要 成功完成最终测试提交后，参与者将看到每个任务三个平均指标的排行榜值。在最终隐藏测试阶段不显示按病例或按图像的得分。 详细结果摘要遵循以下结构： ```text MVAA最终隐藏测试评估 状态：完成 信息：官方隐藏测试评估已完成 任务级平均指标： - 任务1（CT）：平均DSC = ...、平均HD = ...、平均ASD = ... - 任务2（TEE）：平均DSC = ...、平均HD = ...、平均ASD = ... - 任务3（视频/图像）：平均DSC = ...、平均HD = ...、平均ASD = ... Evaluation_Status：1 Runtime_Seconds：... ```

# 问答 ## 最终测试阶段应该提交什么？ 提交包含根目录级 `submission.json` 的 `submission.zip`，其中包含您的 Docker 镜像参考。 ## 是否需要单独上传模型权重？ 否。您的 Docker 镜像必须包含推理所需的所有模型权重和依赖项。 ## Docker 容器是否具有互联网访问权限？ 您的镜像不得在隐藏测试评估期间需要互联网访问。组织者可能会限制或禁用运行时网络访问。 ## 隐藏测试图像安装在哪里？ 隐藏测试输入安装在容器内的 `/input`。 ## 预测结果应写入何处？ 将所有预测 JSON 文件和掩码写入 `/output`。 ## CodaBench 排行榜是否显示最终的 DSC/HD/ASD 分数？ 是的。描述符验证后，私有组织者端工作程序在隐藏测试集上运行提交的镜像，并将官方 Task1/Task2/Task3 的 DSC、HD 和 ASD 指标返回到此排行榜。 ## 为什么我的行可能显示 Evaluation_Status = 0？ `Evaluation_Status = 0` 表示验证、镜像拉取、容器执行、输出检查或评分失败。检查提交日志和详细评分输出中的 `Message` 行。 ## 我的镜像应该支持哪个 GPU？ 官方运行器使用 NVIDIA Tesla V100 32GB。您的镜像必须支持计算能力 `sm_70`。 ## 其他参赛者能否看到我的 Docker 镜像参考？ 否。Docker 镜像参考仅用于组织者端评估，不会在公共排行榜或详细结果页面中显示。如果需要保护代码或权重，请使用私有注册表并向组织者提供拉取访问权限。 ## 是否有参考基线 Docker 镜像？ 是的。参赛者可以拉取公共参考基线镜像以查看预期的基于 Docker 的推理风格： ```bash docker pull db0725/mvaa-baseline-infer:final-20260626 ``` 此镜像仅是参考示例。参赛者可以提交自己的 Docker 镜像，只要它们遵循所需的输入/输出约定。 ## 最终测试阶段允许多少次提交？ 每个团队在最终测试阶段总共可提交最多 10 次。其中，每个团队最多允许 7 次有效提交。 额外的 3 次提交尝试保留用于可能的技术故障，例如描述符格式错误、镜像拉取失败、平台错误或其他意外提交问题。这些失败或无效的尝试可能被组织者自行决定排除在有效提交计数之外。 该限制适用于整个团队，而不是单个 CodaBench 账户。如果同一团队的成员使用多个账户绕过提交限制，该团队可能被取消资格。此规则旨在确保较小团队的公平性。
