Overview

# 概述 欢迎参加**可解释深度伪造检测挑战2026**，这是**ACM Multimedia 2026大赛**中关于图像级深度伪造检测和接地式解释生成的竞赛。 深度伪造检测正在超越简单的人脸交换，转向更丰富的语义操纵。在实际应用中，仅有预测结果是不够的；系统还应该指出支持该决策的视觉证据。 本挑战引入**XPlainVerse**，这是一个大规模基准数据集，用于评估检测和解释的质量。参赛者不仅需要判断图像是否为真实或伪造，还需要提供两个接地式自然语言解释：一个详细的复杂解释和一个非技术性的简单解释。 ## 竞赛任务 本挑战包含两个主要任务： 1. **深度伪造检测**：判断输入图像是真实还是伪造。 2. **深度伪造解释**：使用图像中的可见证据生成支持真伪判断的解释。 对于解释任务，每份提交必须包括： - **复杂解释**：详细、证据充分且具体，旨在供技术或法医审查； - **简单解释**：更短、更清晰、更易理解，同时保持决策的关键原因。 两个解释都应与提交的真伪标签保持一致。 ## CodaBench阶段 本CodaBench平台用于官方结果提交阶段。 - **阶段开启**：2026年5月28日 - **结果提交截止**：2026年6月15日 - **每天提交次数**：1次 - **总提交次数**：10次 一般性挑战信息、公告、注册链接和组织方更新发布在官方挑战网站上： https://explainable-deepfake-detection.github.io/

Data

# 数据 该挑战使用**XPlainVerse**，这是检测和解释任务的官方基准。 公开发布提供了带有配对解释注释和清单元数据的训练和验证数据，以及用于最终评估的测试图像。此挑战的XPlainVerse发布包含**760,000张图像**，其中**450,000张训练图像**、**110,000张验证图像**和**200,000张测试图像**。 测试集为最终评估而发布。测试标签和参考解释不包含在参赛者发布版本中，仅由CodaBench评估系统使用。 ## 数据集结构 发布的数据包组织如下： ```text XPlainVerse/ ├── train/ │ ├── manifest.jsonl │ ├── images/ │ │ ├── fake/ │ │ └── real/ │ ├── complex_explanations/ │ │ ├── fake/ │ │ └── real/ │ └── simple_explanations/ │ └── fake/ ├── val/ │ ├── manifest.jsonl │ ├── images/ │ │ ├── fake/ │ │ └── real/ │ ├── complex_explanations/ │ │ ├── fake/ │ │ └── real/ │ └── simple_explanations/ │ └── fake/ └── test/ └── images/ ``` 训练和验证分割包括一个`images`目录、配对的`complex_explanations`、配对的`simple_explanations`和一个`manifest.jsonl`文件。为真实和虚假图像提供复杂解释，仅为虚假图像提供简单解释。测试分割仅包括发布的图像；标签和参考解释对评估保持隐藏。 ## 清单字段 训练和验证分割的`manifest.jsonl`中的每一行存储了加载图像及其配对解释文件所需的标签和相对路径。 对于训练和验证分割，重要字段包括： - `label`：二进制图像标签，虚假或真实； - `image_path`：图像文件的相对路径； - `complex_explanation_path`：配对复杂解释JSON的相对路径； - `simple_explanation_path`：配对简单解释JSON的相对路径。 ## 隐藏测试集 CodaBench最终阶段使用发布的测试图像与隐藏的真实标签和参考解释。这些参考不包含在参赛者发布版本中，仅由CodaBench工作人员使用。 对于每个测试图像，提交文件中的`id`字段必须完全匹配测试集中的图像文件名（包括文件扩展名），例如`example.png`。

Evaluation

# 评估 排行榜报告检测性能和解释质量。所有显示的指标中，分数越高越好。 ## 检测 检测在隐藏的最终测试集上使用二元图像标签进行评估： - `0`：真实/原真图像； - `1`：假图像/生成或被篡改的图像。 排行榜报告： - **检测宏观F1分数**：主要检测分数； - **检测准确度**； 检测在完整的官方测试集上进行评估。 ## 复杂解释 复杂解释追踪评估提交的解释是否捕捉了与参考解释相同的重要视觉证据。 在此CodaBench评分包中，复杂解释使用提交的复杂解释和隐藏参考复杂解释之间的**BERTScore F1**进行评估。 BERTScore模型为： ```text microsoft/deberta-xlarge-mnli ``` 基线重缩放已禁用。 对于排行榜评分，复杂解释BERTScore在测试集的10,000样本子集上计算。测试阶段关闭后，我们将为**前5个队伍**计算剩余的复杂解释指标：**复杂实体F1**和**复杂证据F1**，用于最终报告和分析。完整的复杂解释分数为： ```text complex_overall_score = 0.3 * complex_bert_f1 + 0.4 * complex_entity_f1 + 0.3 * complex_evidence_f1 ``` ## 简单解释 简单解释追踪旨在同时奖励两个方面：忠实性和简洁性。好的简单解释应该保留参考中的核心原因，同时以更短、更易理解的形式表达。 在此CodaBench评分包中，简单解释使用以下方法进行评估： - **简单BERTScore F1**与隐藏的简单参考对比； - **SLE分数**使用工作人员的离线SLE检查点； - **简单SLE标准化**，通过将原始SLE裁剪为`[-1, 4]`并将其映射到`[0, 1]`得到； - **简单整体分数**： ```text 0.7 * simple_bert_f1 + 0.3 * simple_sle_norm ``` 对于排行榜评分，简单解释指标在相同的10,000样本解释评估子集上计算。 ## 总体解释分数 显示的解释分数将复杂和简单两个追踪在相同的10,000样本解释评估子集上相结合： ```text explanation_score = (complex_bert_f1 + simple_overall_score) / 2 ``` ## 失败或不完整的提交 缺失的文件在相应的指标组中得零分。如果提交文件有效但遗漏某些必需行，这些缺失样本在相关指标中被评分为零。格式错误的JSON、重复的ID、无效的标签、空解释或未知ID可能导致评分失败。

Submission

# 提交要求 提交单个 `.zip` 文件，包含官方测试样本的模型输出。 ```text submission.zip ├── detection.jsonl ├── complex.jsonl └── simple.jsonl ``` 每一行必须是一个有效的 JSON 对象。**`id` 值必须完全匹配发布的测试图像文件名，包括扩展名**。 ## 检测文件 `detection.jsonl` 应包含每个最终测试图像的一行： ```json {"id": "sample1.png", "pred_label": 1} ``` `pred_label` 必须为： - `0` 表示真实； - `1` 表示伪造。 ## 复杂解释文件 `complex.jsonl` 应包含解释评估子集的复杂解释： ```json {"id": "sample1.png", "complex_explanation": "基于可见图像证据的详细解释。"} ``` 复杂解释应该是基于可见图像证据的详细解释，例如伪影、几何问题、纹理不一致、文本错误、光线不匹配或物体交互。 ## 简单解释文件 `simple.jsonl` 应包含同一解释评估子集的简单解释： ```json {"id": "sample1.png", "simple_explanation": "针对该图像的简单解释。"} ``` 简单解释应该是更短且更易理解的解释，保留决策的关键原因，但不包含复杂解释中的每个低级细节。 ## 提交限制 每个团队最多可以进行： - **每天 1 次提交**； - **最终阶段总共 10 次提交**。 ## 排行榜条目 每个团队在排行榜上只能有一个公开条目。团队可以使用 CodaBench 中的提交控件选择要在排行榜上显示的提交运行。 ## 上传前验证 起始工具包包含格式检查器。在提交前在本地运行： ```bash python validate_submission.py submission.zip ``` 验证器仅检查公开格式规则。
