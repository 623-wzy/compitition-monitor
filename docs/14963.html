竞赛描述

DeepPhenoTree - Apple Edition 数据集（包含 1,438 张苹果树 RGB 图像，采集自瑞士、比利时、西班牙和意大利，2025 年）包含 22,500 个手动标注的边界框，涵盖三个物候阶段：开花期、果实小球期和果实期。 开发并基准测试目标检测模型，该模型能够自动定位图像中的三个物候阶段。提交的结果将使用平均精度均值（mAP@0.5）进行评估，这是检测挑战的标准指标。 挑战赛的结果将在 ECCV 会议（2026 年 9 月 8-13 日）期间的 CVPPA 研讨会（9 月 8-9 日）中呈现，会议地点为瑞典马尔默。

DeepPhenoTree – ECCV 2026 CVPPA Data Challenge

# DeepPhenoTree – ECCV 2026 CVPPA数据挑战赛 本挑战赛的目标是开发能够检测代表苹果树物候阶段的对象的模型。 ## 为什么要进行物候检测挑战赛？ 物候是植物发育的关键指标，从休眠期到收获期，在果园管理中起着核心作用。然而，物候评估仍主要依靠人工进行，这使得该过程耗时且难以规模化。本挑战赛在真实条件下引入了一个标准化的物候对象检测基准。数据集从四个欧洲苹果试验果园基地收集[1]，在多个基地、采集条件和物候阶段中捕捉了强烈的变异性，同时受益于受控照明条件。 该任务特别具有挑战性，原因包括： - 不同基地和采集条件之间的变异性， - 树冠内不同的对象大小和遮挡现象， - 相邻物候阶段之间的高类内相似性 所有图像均使用标准化成像平台在受控照明条件下采集，确保各基地之间视觉质量的一致性，同时保留真实的开放田间变异条件。本挑战赛的一个关键目标是公开推广能够跨基地和条件泛化的模型。 ## 挑战赛概述 给定苹果树的输入图像，参赛者需要设计一个模型来预测与德国农业协会、联邦植物品种办公室和化学工业协会(BBCH)[2]相关的对象的边界框，包括三个主要物候阶段：从休眠期到花期(BBCH 00-69)、小果期(BBCH 71-77)和果实期(BBCH 81-89)。真实标注包括手动标记的边界框及相应的主要物候阶段。真实标注框以YOLO格式提供(类别0 = 花期，1 = 小果期，2 = 果实期)。 模型在每次提交时自动进行评估。 在开发阶段，参赛者在公开测试集上获得实时评估分数，并可迭代改进其模型。 在盲评阶段，对单独的私有测试集进行预测，最终分数仅在竞赛结束时公布，以确保排名公平。 ## 贡献 ### 论文(审查中) Metuarea Herearii, Ousseini Hamza Abdoul-Djalil, Guerra Walter, Zuffa Francesca, Panzeri Francesco, Patocchi Andrea, Lozano Lidia, Van Hoye Shauny, Laurens François, Labrosse Jeremy, Rasti Pejman & Rousseau David. Challenge and Timeline

# 挑战与时间表 ## 任务 给定苹果树的输入图像，参赛系统必须为图像中的每个实例输出一个边界框和相应的物候阶段标签。 * **输入：** 苹果树的RGB图像（在`images/`文件夹中提供）。 * **输出：** 与图像同名的`.txt`文件，每行包含一个检测结果： ```txt <class_id> <x_center> <y_center> <width> <height> <confidence> ``` 其中： - **class_id**（0 = 开花，1 = 小果实，2 = 果实） - **x_center, y_center, width, height**（边界框坐标，归一化到图像宽度/高度，YOLO格式，每个检测一行） - **confidence**是模型对给定边界框输出的置信度（如果未提供，默认为1） 数据集包含多实例图像，对应于树的不同部分处于不同的物候阶段。系统必须对每个图像中所有可检测的实例进行预测。 每个对象应该仅被检测一次（无高重叠的重复检测）。详见**竞赛指标**部分。 ## 提交格式 遵循下述提交指南。 ### 1. 模型开发 下载开发阶段的数据。可自由使用训练/验证集划分、外部数据或预训练模型。 可能会提供基准实现和其他资源以便于重现性和基准测试。 ### 2. 预测生成 对于测试集中的每个图像，参赛者必须使用Python文件以YOLO格式生成预测文件。Python文件必须定义一个名为Model的类。该类必须能被评估系统加载，并暴露一个具有以下结构的预测方法： ``` class Model: def __init__(self, ...): ... def predict(self, image_path: str): ... ``` 预测： - 输入：测试图像的路径 - 输出：YOLO格式的字符串列表，如下所示： ``` class_id x_center y_center width height confidence ``` 其中`class_id`是类索引，`x_center y_center width height`是边界框坐标（范围[0,1]），按图像宽高归一化，`confidence`是置信度（范围[0,1]）。 ### 3. 打包提交 创建名为`deepphenotree_challenge2026.zip`的zip存档，结构如下： ``` deepphenotree_challenge2026/ ├─ model.py ├─ weights.pth（或其他格式） ├─ params.yml（如模型需要） └─ README.md （可选的简短描述） ``` 参赛者可使用标准Python深度学习库。推理所需的任何非标准依赖必须在README文件中明确说明。 在**Codabench**竞赛页面上传zip文件。 * 为测试集中的**所有图像**提供预测。 * 所有坐标必须是0到1之间的浮点数。 * 文件必须使用UTF-8编码和Unix行尾。 ## 竞赛指标 提交内容使用**平均精度均值(mAP)** IoU 0.5自动评分，这是目标检测竞赛的标准指标。 * **mAP@0.5:0.95** - 主排行榜得分（IoU阈值0.5-0.95的平均值）。 * **mAP@0.5** - 次要得分。 评估流程： 1. 对于每个图像，使用匈牙利算法将预测与真实框匹配。 2. 当IoU ≥ 0.5且类别匹配时，计数为真正例。 3. 计算每个类别的精确率-召回率曲线，然后取平均值得到mAP。 ## 时间表 | 日期（2026年） | 里程碑 | |-------------|-----------| | **4月15日** | Codabench竞赛页面上线；排行榜开始更新 | | **4月15日** | 发布训练集 | | **6月26日** | 开发阶段关闭；排行榜不再更新 | | **6月26日-7月5日** | 上传最终预测代码/Docker镜像的截止日期 | | **7月5-6日** | 发布最终排行榜；宣布获胜者 | | **7月10日** | 论文提交截止日期（EasyChair） | | **7月14日** | ... |

FAQ

# 常见问题 ❓ 我可以使用外部数据或预训练模型吗？ 可以。参赛者被允许使用外部数据集或预训练模型（例如ImageNet、COCO或自监督模型），除非另有明确说明。所有训练流程必须保持可重现性，参赛者需要在README.md中清楚地记录所使用的任何外部资源。 ❓ 我需要使用提供的训练/验证数据划分吗？ 不需要。提供的数据划分是推荐的，但不是强制的。参赛者可以自由定义自己的训练/验证数据划分。 ❓ 我的模型在什么环境中进行数据导入/评分？ 评估在配备NVIDIA V100（32GB）GPU的专用运行器上执行。 时间限制：整个推理过程（加载模型和处理所有测试图像）必须在30分钟内完成。超出此限制的提交将自动超时。 环境：所有脚本在受控的Docker容器内执行以确保可重现性。您可以在此处访问官方基础镜像：`registry.forge.inrae.fr/imhorphen/deepphenotree/cvppa-2026-base:v1`。 软件堆栈：要检查预装库及其确切版本，可以运行以下命令： ```bash podman run --rm \ registry.forge.inrae.fr/imhorphen/deepphenotree/cvppa-2026-base:v1 \ pip freeze ``` ❓ 如果我的提交不遵循所需格式会怎样？ 提交内容自动进行评估。任何不遵循所需结构或界面的提交可能在数据导入期间失败，将不被评估。强烈建议参赛者在上传前验证其提交。 ❓ 我可以使用开发或测试数据进行训练吗？ 不可以。虽然测试标签无法访问，但开发和盲测集不得用于训练或模型调优。 更多问题：请联系 **challenge2026-deepphenotree@inrae.fr**
