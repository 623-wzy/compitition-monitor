EBMV 2026 LOW-LEVEL IMAGING CHALLENGE: EVENT-GUIDED LOW-LIGHT VIDEO DEPTH ESTIMATION ON SDE DATASET

ID: 16281 创建者: ethanliang99 创建时间: 2026-05-13 参与人数: 14 提交次数: 0 在 Codabench 查看 ↗

竞赛描述

本挑战赛重点关注SDE数据集上的事件引导低光视频深度估计。参赛者需要开发算法，在事件相机信号的辅助下，从低光视频序列中估计准确的深度图。 该挑战赛的目标是促进在具有挑战性的光照条件下的鲁棒深度估计，在这些条件下，传统的基于帧的方法通常会受到运动模糊、低信噪比和纹理可见性差的影响。提交的方案将使用隐藏测试集上的定量深度估计指标进行评估。 本挑战赛作为EBMV 2026的一部分组织举办，面向从事低级视觉、基于事件的视觉、视频增强以及单目或多模态深度估计研究的科研人员。

Overview

<h2>EBMV 2026 低光成像挑战赛</h2> <h3>事件引导的低光视频深度估计——基于SDE数据集</h3> <p> 本挑战赛聚焦于低光场景下的事件引导视频深度估计。 参赛者需要结合低光RGB帧和事件流来预测每个场景的稠密深度图。 </p> <p> 基准在场景/视频级别组织，以保持帧间的时间一致性。 </p> <h3>任务</h3> <p> 给定低光视频序列及其对应的事件相机数据，参赛者应为每个目标场景估计一个配准的稠密深度序列。 </p> <h3>关键词</h3> <ul> <li>事件相机</li> <li>低光视频</li> <li>视频深度估计</li> <li>深度估计</li> <li>低级成像</li> </ul>

Data

<h2>数据集</h2> <p> 竞赛数据集是与研讨会对齐的DVD事件引导视频深度估计数据的版本。 它将每个场景打包到规范的文件夹树中，对齐低光RGB帧、事件切片、场景级事件流和深度监督。 </p> <h3>数据集下载</h3> <p> 数据集托管在Hugging Face上： </p> <p> <a href="https://huggingface.co/datasets/Ethanliang99/ECCV_Event_Video_Depth_Estimation/tree/main" target="_blank"> https://huggingface.co/datasets/Ethanliang99/ECCV_Event_Video_Depth_Estimation/tree/main </a> </p> <h3>数据集结构</h3> <p> 官方划分直接编码在顶级目录结构中： </p> <pre><code>train/ &lt;scene_name&gt;/ low/ normal/ val/ &lt;scene_name&gt;/ low/ normal/ test/ &lt;scene_name&gt;/ low/ normal/ </code></pre> <p> 每个场景目录包含对齐的低光输入和深度目标。还提供了<code>manifest.json</code>文件，包含发布和划分元数据。 </p> <h3>场景布局</h3> <pre><code>&lt;scene_name&gt;/ low/ &lt;timestamp&gt;.png &lt;timestamp&gt;.npz lowlight_event.npz normal/ &lt;timestamp&gt;.png depth.npz manifest.json </code></pre> <h3>输入</h3> <ul> <li><code>low/</code>下的低光RGB帧</li> <li><code>low/</code>下的逐帧事件切片</li> <li>存储为<code>lowlight_event.npz</code>的场景级事件流</li> </ul> <h3>输出/监督</h3> <ul> <li>存储为<code>normal/depth.npz</code>的对齐深度序列</li> </ul> <h3>重要说明</h3> <ul> <li>数据集按场景/视频数量划分，而不是按帧数划分。</li> <li>文件夹名称本身指示划分分配。</li> <li>研讨会版本中的事件切片已修剪，以避免重复计算重叠事件。</li> <li>在训练或评估时间模型时，请保持场景级时间顺序完整。</li> </ul>

Submission

<h2>提交格式</h2> <p> 参赛者应为每个测试场景提交一个预测深度序列。 预测深度应与相应场景的帧顺序保持对齐。 </p> <h3>推荐目录结构</h3> <pre><code>submission/ &lt;scene_name&gt;/ normal/ depth.npz </code></pre> <h3>深度文件格式</h3> <p> 每个<code>depth.npz</code>文件应包含相应场景的预测深度数组。 建议使用形状为<code>[T, H, W, 1]</code>的4D张量，以兼容现有的DVD风格工具。 评估脚本也可能支持形状为<code>[T, H, W]</code>的3D张量。 </p> <h3>提交要求</h3> <ul> <li>提交所有必需测试场景的预测结果。</li> <li>保持原始场景名称。</li> <li>保持每个场景内的时间帧顺序。</li> <li>不要在提交物中包含真实标签文件。</li> <li>在上传前将<code>submission/</code>文件夹压缩为zip文件。</li> </ul>

Evaluation

<h2>评估</h2> <p> 将通过将预测的深度序列与隐藏的地面实况深度标注进行比较来评估提交。 </p> <h3>指标</h3> <p> 将报告以下指标： </p> <ul> <li>AbsRel / <code>abs_relative_difference</code>（绝对相对差异）</li> <li>RMSE / <code>rmse_linear</code>（均方根误差）</li> <li>delta1 / <code>delta1_acc</code></li> <li>delta2 / <code>delta2_acc</code></li> <li>delta3 / <code>delta3_acc</code></li> <li>SILog / <code>silog_rmse</code>（尺度不变对数RMSE）</li> </ul> <h3>排名指标</h3> <p> 最终排名指标将在官方评估阶段前宣布。 在启动阶段，我们建议使用 <strong>AbsRel</strong> 作为主要排名指标，并将其余指标作为诊断指标进行报告。 </p> <h3>评估协议</h3> <ul> <li>按场景逐个评估预测。</li> <li>每个预测的深度序列应与对应的输入序列在时间上对齐。</li> <li>无效或缺失的预测可能导致评估失败或受到惩罚。</li> </ul>
