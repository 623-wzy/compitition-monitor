<div align="center"> <h1><strong>第二届低层次视觉前沿研讨会 (LoViF) @ ECCV 2026</strong></h1> </div> <h2>&nbsp;</h2> <h1>真实世界全能图像复原挑战赛</h1> <h3>&nbsp;</h3> <h2>数据概览</h2> <p align="justify">本挑战赛的数据集由<a href="https://github.com/House-Leo/FoundIR">FoundIR</a>和LoViF组织委员会提供。该数据集专门设计用于基准测试低层次视觉的前沿技术，涵盖五类常见真实世界退化图像：<strong>模糊、低光、雾霾、雨水和雪</strong>。数据集包含用于训练、验证和测试的多样化场景。具体来说，每种退化包含4,900对配对的LQ–GT图像（512×512）用于训练，100张图像用于验证，100张图像用于测试。</p> <p align="justify">在本竞赛中，数据集由以下部分组成：</p> <ul> <li><strong>训练数据：</strong>24,500对退化图像及其对应的真值参考。</li> <li><strong>验证数据：</strong>500张退化图像，用于开发阶段的性能调优。</li> <li><strong>测试数据：</strong>500张退化图像，用于最终性能评估。</li> </ul> <p align="justify">请注意，在验证集和测试集中，索引为<strong>0001-0100的图像对应模糊、0101-0200对应低光、0201-0300对应雾霾、0301-0400对应雨水、0401-0500对应雪</strong>。</p> <h3>&nbsp;</h3> <h2>数据访问</h2> <p align="justify"><strong>通过访问数据，参与者隐含同意LoViF挑战赛的条款和条件。</strong></p> <h3>&nbsp;</h3> <ul> <ul> <li><strong>开发阶段/学习：</strong></li> <ul> <li>训练数据：<a href="https://drive.google.com/file/d/1DA28a64FEGRM2bd0RWSLsgZcDybcWWO2/view?usp=sharing">[谷歌云盘]</a>。参与者允许使用公开可用的外部数据集进行训练，但数据源必须在说明书中明确记录。有关训练数据的更多详情可在<a href="https://github.com/House-Leo/FoundIR">FoundIR</a>中找到。</li> <li>验证数据（仅输入图像）：<a href="https://drive.google.com/file/d/1r488W9gwk-fy4BkwOGeqp71U0hk_s_um/view?usp=sharing">[谷歌云盘]</a>。</li> <li>测试数据（仅输入图像）：[待定]</li> </ul> </ul> </ul> <h3>&nbsp;</h3> <h2>最终测试阶段提交指南（<span style="text-decoration: underline;">说明书可在<a href='https://www.overleaf.com/read/zpgpfqfqrgrb#fb6d1d'>[Overleaf]</a>中找到</span>）</h2> <p>最终提交应通过电子邮件发送至组织者（详见"组织者"选项卡以获取联系方式）。</p> <p>邮件标题应为：<strong>[LoViF2026] - evaluation

<div align="center"> <h1><strong>第2届低级视觉前沿研讨会(LoViF) @ ECCV 2026</strong></h1> </div> <h2>&nbsp;</h2> <h1>真实世界全能图像复原挑战赛</h1> <h3>&nbsp;</h3> <h3>评估</h3> <p><span>评估包括将复原/生成的图像与参考真实图像进行比较。<span>请确保所有图像使用统一的模型进行处理，以保持不同测试场景的一致性。</span></span></p> <p align="justify">为了全面评估低级视觉前沿领域的性能，我们采用基于保真度和基于感知的指标，包括<strong>PSNR</strong>、<strong>SSIM</strong>和<strong>LPIPS</strong>。</p> <p align="justify">由于本挑战赛的目标是探索<strong>稳健且强大的全能图像复原模型</strong>，参赛者需要使用一个复原网络来处理所有类型的图像退化并获得最终结果。 具体来说，参赛者应确保最终提交仅包含<strong>一个训练好的图像复原模型参数文件（例如.pt、.pth）</strong>，这有利于组织方对实验结果的可重复性验证。 需要说明的是，允许使用现有的预训练网络（例如VAE、CLIP）进行特征提取，这些不计入上述模型参数文件。 </p> <p align="justify">最终排名将由平衡像素级精度和感知质量的综合得分决定。初始结果的加权公式为：<br> <strong>Final_Score = PSNR (Y) + 10 * SSIM (Y) - 5 * LPIPS</strong>。</p> <p>提交结果时，请遵循以下说明：</p> <ol type="1"> <li>为确保评估的公平性，参赛者建议使用以下提供的代码将模型输出图像保存为JPG格式。 <pre><code class="language-python"> from PIL import Image img = Image.open(img_path) img.load() if img.mode != "RGB": img = img.convert("RGB") save_kwargs = dict(format="JPEG", quality=96, subsampling=0, optimize=True) img.save(out_path, **save_kwargs) </code></pre> </li> <li><strong>处理</strong>输入图像，并为你的方法生成的输出图像结果保留相同的名称（示例：对于名为"0001.jpg"的输入文件，输出文件应为"0001.jpg"）。注意输出图像应具有与输入图像相同的尺寸。<a href='https://drive.google.com/file/d/1J_cHUcbalSlACQVA2JUQXrCglRhJqdYz/view?usp=sharing'>[提交示例]</a>。 </li> <li>确保你的提交包含方法的简短描述（事实说明书）。</li> <li>在截止日期前将你的结果提交到Codabench服务器。</li> </ol> <p>&nbsp;</p> <p align="justify">有关研讨会主题的更多详情，如偏好优化或评估中的代理系统，请访问我们的官方网站：<a href="https://lovif-eccv2026-workshop.github.io/">https://lovif-eccv2026-workshop.github.io/</a></p>
