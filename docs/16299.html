Overview

# 赛道1：情感感知艺术图像生成 ## 概述 给定描述语义内容、艺术风格和期望情感状态的多模态提示词，赛道1的目标是生成满足全部三个条件的图像。 该赛道评估生成模型是否能够保留请求的视觉内容，同时与指定的艺术风格和目标情感状态保持一致。 ## 问题陈述 参赛者获得一组图像描述文字。 对于每个描述，参赛者必须生成一个艺术图像，该图像同时满足所描述的语义内容、艺术风格和情感条件。 ## 科学挑战 参赛者需要开发能够将风格与情感分离开来的模型。成功的模型应学会巧妙地操纵视觉属性以实现混合状态，例如冷静的表现主义绘画。 这要求模型通过包括构图、笔触、色彩、线条和光线等视觉属性来推理艺术风格和情感表现。 ## 评估指标 赛道1提交内容将使用以下指标进行评估： - 弗雷歇初始距离（FID），用于评估图像质量和与参考艺术作品图像的分布级相似度。 - 属性对齐得分（AAS），用于评估提示词与图像在内容、风格和艺术属性方面的对齐程度。 最终赛道1得分将FID和AAS以相等权重相结合。

Submission

# 提交说明 参赛者必须上传遵循规定的 Track 1 格式的 `submission.zip` 文件。 不要在 zip 文件内包含额外的父目录。所需文件应直接出现在 `submission.zip` 的顶层。 参赛者必须提交包含以下内容的 `submission.zip` 文件： ```text submission.zip +-- submission.json `-- images/ +-- track1_0001.jpg +-- track1_0002.jpg `-- ... ``` `submission.json` 文件必须遵循以下格式： ```json [ { "sample_id": "track1_0001", "path": "images/track1_0001.jpg" }, { "sample_id": "track1_0002", "path": "images/track1_0002.jpg" } ] ``` 每个生成的图像文件名必须与其对应的 `sample_id` 完全匹配。 ## 上传规则 - 仅上传所需的 `submission.zip` 文件。 - 不要上传完整项目文件夹或源代码，除非另有说明。 - 确保所有 `sample_id` 值与官方测试集相匹配。 - 确保 `submission.json` 中的所有路径与 zip 压缩包内的文件相匹配。 - 无效的 JSON、缺失样本、缺失文件或文件名不匹配可能导致提交验证失败。

Evaluation

# 评估 ## 赛道1：情感感知艺术图像生成 赛道1的提交作品将使用Frechet初始距离(FID)和属性对齐分数(AAS)进行评估。 ## FID FID测量生成图像集与参考艺术图像集之间的分布级相似性。 FID越低，性能越好。 ## AAS AAS评估每个生成图像与其对应标题之间的提示级对齐程度。它包括三个方面： - 内容对齐：生成的图像是否正确反映提示中描述的主要视觉内容。 - 风格对齐：生成的图像是否反映所需的艺术风格。 - 属性对齐：生成的图像是否与标题中指定的艺术属性一致，如构图、笔法、色彩、线条或光线等适用方面。 AAS将使用标准化的大语言模型辅助多模态评估协议进行评估。每个生成的图像将与其对应的提示一起在固定的评估提示、预定义的评分标准和一致的推理设置下进行评估。最终的AAS将通过聚合三个对齐方面和所有测试样本的分数来计算。 赛道1的最终分数将通过以相等权重结合FID和AAS来计算：50% FID和50% AAS。由于较低的FID表示更好的性能，FID在与AAS结合之前将被归一化或转换为分数。 ## 无效内容 任何试图操纵评估器的行为，如在评估器中包含针对评估器的指令而非有效任务响应，将被视为无效内容，可能导致分数扣除或取消资格。
