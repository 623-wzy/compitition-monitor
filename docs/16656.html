overview

<div align="center"> <h1><strong>第2届低级视觉前沿工作坊(LoViF) @ ECCV 2026</strong></h1> </div> <h2>&nbsp;</h2> <h1>超低比特率图像压缩挑战赛</h1> <h2>&nbsp;</h2> <h2>重要时间节点</h2> <ul> <li>2026.05.27 — 发布训练数据</li> <li>2026.05.27 — 发布验证数据</li> <li>2026.05.27 — 验证服务器上线</li> </ul> <ul> <li><strong>2026.07.11 — 解码器可执行文件提交截止(最大文件大小：4 GB)</strong></li> <li><strong>2026.07.12 — 发布最终测试数据</strong></li> <li><strong>2026.07.15 — 测试结果提交截止</strong></li> <li><strong>2026.07.15 — 信息表单提交截止</strong></li> <li><strong>2026.07.20 — 向参赛者发布初步测试结果</strong></li> <li><strong>2026.07.20 — 工作坊论文提交截止</strong></li> <li>2026.09.xx — LoViF工作坊和挑战赛、结果发布及颁奖仪式(ECCV 2026)</li> </ul> <p> </p> <h2>挑战赛概述</h2> <p align="justify"> <a href="https://lovif-eccv2026-workshop.github.io/">第2届低级视觉前沿工作坊(具有生成式人工智能、偏好优化和代理系统)</a>将于2026年9月与ECCV 2026同期召开。 </p> <p align="justify"> 图像压缩正在快速发展，朝着更加感知导向的视觉编码范式演进。传统编码器主要关注逐像素保真度，这在超低比特率下往往导致严重的模糊、分块伪影和细节纹理丧失。最近，先进的基于学习的方法，特别是那些利用强大生成先验的方法，在合成感知逼真细节和在极限比特率预算下改进视觉质量方面展示了强大潜力。 </p> <p align="justify"> 本挑战赛聚焦于超低比特率图像压缩，旨在推进感知重建、视觉质量和极限比特率约束下紧凑视觉表示的边界。参赛者被鼓励探索创新的压缩框架，包括融入先进生成先验(例如扩散模型、GANs、自回归模型等)的方法，以及其他先进的基于学习的技术。我们欢迎来自学术界和产业界的研究人员和从业者对标他们的最新算法，并为下一代图像压缩系统的发展做出贡献。 </p> <p> </p> <h2>本次竞赛的动机</h2> <p align="justify"> 虽然经典图像压缩已经实现了高编码效率，但传统方法在超低比特率下往往存在严重的模糊、分块和纹理退化问题。这主要是因为标准优化指标(如PSNR和MS-SSIM)优先考虑逐像素保真度而非感知真实性，这可能不能完全符合人类视觉偏好。 </p> <p align="justify"> 基于学习的图像生成的最近进展为超低比特率压缩开辟了新机遇。特别是，生成先验和感知驱动的重建策略在从高度紧凑的表示中恢复逼真纹理和改进视觉质量方面展现了强大能力。然而，在极限比特率约束下平衡感知质量、语义一致性和编码效率仍然是一个具有挑战性的问题。 </p> <p align="justify"> 为了促进该领域的研究，LoViF挑战赛引入了一个专门的基准测试，聚焦于<strong>超低比特率图像压缩</strong>。我们鼓励参赛者探索先进的感知重建技术和创新编码策略，在改进人类视觉体验的同时保持竞争力的压缩性能。最终，本竞赛旨在建立一个实用且前瞻性的基准测试。 </p>

data

<div align="center"> <h1><strong>第2届低阶视觉前沿研讨会(LoViF) @ ECCV 2026</strong></h1> </div> <h2>&nbsp;</h2> <h1>超低比特率图像压缩挑战赛</h1> <h2>&nbsp;</h2> <h2>数据概览</h2> <p align="justify"> 本挑战赛的数据集由开源高分辨率图像数据集的精选样本和从互联网收集的高质量图像组成。该数据集专门为超低比特率图像压缩的前沿基准而精选，具有多样化的真实场景和高质量视觉内容。在本次竞赛中，数据集包括： </p> <ul> <li> <strong>训练数据：</strong> 提供共计14,136张高质量图像用于训练图像压缩模型。鼓励参赛者使用此数据集。允许使用其他公开数据集，但所有外部数据源必须在说明书中清楚地标注。 </li> <li> <strong>验证数据：</strong> 提供共计100张高质量图像，用于开发阶段的性能调优。 </li> <li> <strong>测试数据：</strong> 另外提供共计100张高质量图像，用于测试阶段的最终评估。 </li> </ul> <h2>数据获取</h2> <p align="justify"> <strong> 参赛者通过访问数据，即表示隐式同意挑战赛的条款和条件。 </strong> </p> <ul> <li> <strong>训练数据：</strong> <ul> <li> <a href="https://huggingface.co/datasets/happyk/image_data/resolve/main/dataset_train.zip"> 训练数据集 </a> (14,136张高质量图像，来自Flickr2K训练集[1]、DIV2K训练集[2]、CLIC2020训练和验证集[3]、CLIC2021测试集[4]、LSDIR[5]) </li> </ul> </li> <li> <strong>开发阶段：</strong> <ul> <li> <a href="https://huggingface.co/datasets/happyk/image_data/resolve/main/dataset_val.zip"> 验证数据 </a> (100张高质量图像，来自DIV2K测试集[2]、CLIC2020测试集[3]、Kodak[6]) </li> </ul> </li> <li> <strong>测试阶段：</strong> <ul> <li> 测试数据： 将在测试阶段开始时发布。 </li> </ul> </li> </ul> <strong>参考文献</strong> <ol> <li> Agustsson, E., &amp; Timofte, R. <i>NTIRE 2017单图像超分辨率挑战赛：数据集和研究。</i> CVPRW 2017。 </li> <li> Timofte, R., Agustsson, E., ... &amp; Zhang, L. <i>NTIRE 2017单图像超分辨率挑战赛：方法和结果。</i> CVPRW 2017。 </li> <li> Toderici, G., Theis, L., ... &amp; Timofte, R. <i>CLIC 2020：学习图像压缩挑战赛。</i> 2020。 </li> <li> Toderici, G., Theis, L., ... &amp; Timofte, R. <i>CLIC 2021：学习图像压缩挑战赛。</i> 2021。 </li> <li> Li, Y., Zhang, K., ... &amp; Van Gool, L. <i>LSDIR：大规模图像复原数据集。</i> CVPR 2023。 </li> <li> Eastman Kodak Company. <i>柯达图像数据库。</i> 1993。 </li> </ol>

evaluation

<div align="center"> <h1><strong>第2届低层次视觉前沿研讨会（LoViF）@ ECCV 2026</strong></h1> </div> <h2>&nbsp;</h2> <h1>超低比特率图像压缩竞赛</h1> <h2>&nbsp;</h2> <h2>开发阶段</h2> <p align="justify"> 在开发阶段，提交的方案将在超低比特率约束下使用保真度导向和感知导向的指标进行评估。评估指标包括PSNR、MS-SSIM、LPIPS、DISTS和比特率（BPP）。PSNR和MS-SSIM均在RGB域中计算。 </p> <p align="justify"> 本竞赛要求参赛者在<strong>单一比特率工作点</strong>提交结果。整个数据集的平均比特率约束为： </p> <p align="center"> <code>average_bpp ≤ 0.008</code> </p> <p align="justify"> 违反比特率约束的提交将被视为无效，不会出现在排行榜上。 </p> <p align="justify"> 开发阶段的排行榜排名将由以下加权评分决定： </p> <p align="center"> <code> Final_Score = PSNR + 10 × MS_SSIM + 20 × (1 - LPIPS) + 25 × (1 - DISTS) </code> </p> <h2>测试阶段</h2> <p align="justify"> 请注意，参赛者必须在最终评估中使用<strong>代码提交阶段提交的对应模型检查点</strong>。 </p> <p align="justify"> 最终排名将主要通过<strong>主观评估</strong>确定。提交的方法将从重建保真度和感知质量两个角度进行评估，以更好地反映在超低比特率约束下的人类视觉偏好。 </p> <p align="justify"> 详细的提交说明和所需文件格式请参阅提交页面。 </p> <h2>最终测试阶段 - 初步结果 [待公布]</h2> <p align="justify"> 评估完成后，初步结果将在此发布。 </p>

submission
