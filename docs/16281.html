EBMV 2026 LOW-LEVEL IMAGING CHALLENGE: EVENT-GUIDED LOW-LIGHT VIDEO DEPTH ESTIMATION ON SDE DATASET

ID: 16281 创建者: ethanliang99 创建时间: 2026-05-13 参与人数: 8 提交次数: 0 在 Codabench 查看 ↗

竞赛描述

该挑战赛专注于SDE数据集上的事件引导低光视频深度估计。参与者需要开发算法，在事件相机信号的辅助下，从低光视频序列中估计精确的深度图。 该竞赛的目标是在具有挑战性的照明条件下推进鲁棒深度估计，因为传统的基于帧的方法通常会受到运动模糊、低信噪比和纹理可见性差的困扰。提交的方案将使用隐藏测试集上的定量深度估计指标进行评估。 该挑战赛作为EBMV 2026的一部分进行组织，面向从事低级视觉、事件视觉、视频增强以及单目或多模态深度估计研究的学者。

Overview

<h2>EBMV 2026 低光级成像挑战赛</h2> <h3>事件引导的低光视频深度估计在SDE数据集上的应用</h3> <p> 本挑战赛聚焦于低光场景中的事件引导视频深度估计。 参赛者需要使用低光RGB帧和事件流来预测每个场景的密集深度图。 </p> <p> 基准测试在场景/视频级别进行组织，以保持帧间的时间一致性。 </p> <h3>任务</h3> <p> 给定低光视频序列和相应的事件相机数据，参赛者应为每个目标场景估计一个对齐的密集深度序列。 </p> <h3>关键词</h3> <ul> <li>事件相机</li> <li>低光视频</li> <li>视频深度估计</li> <li>深度估计</li> <li>低光级成像</li> </ul>

Data

<h2>数据集</h2> <p> 竞赛数据集是经过工作坊对齐处理的DVD事件引导视频深度估计数据。它将每个场景打包成标准文件夹结构，对齐了低光RGB帧、事件切片、场景级事件流和深度监督信息。 </p> <h3>数据集下载</h3> <p> 数据集托管在Hugging Face上： </p> <p> <a href="https://huggingface.co/datasets/Ethanliang99/ECCV_Event_Video_Depth_Estimation/tree/main" target="_blank"> https://huggingface.co/datasets/Ethanliang99/ECCV_Event_Video_Depth_Estimation/tree/main </a> </p> <h3>数据集结构</h3> <p> 官方数据划分直接编码在顶层目录结构中： </p> <pre><code>train/ &lt;scene_name&gt;/ low/ normal/ val/ &lt;scene_name&gt;/ low/ normal/ test/ &lt;scene_name&gt;/ low/ normal/ </code></pre> <p> 每个场景目录包含对齐的低光输入和深度目标。还提供了<code>manifest.json</code>文件，包含版本和划分元数据。 </p> <h3>场景布局</h3> <pre><code>&lt;scene_name&gt;/ low/ &lt;timestamp&gt;.png &lt;timestamp&gt;.npz lowlight_event.npz normal/ &lt;timestamp&gt;.png depth.npz manifest.json </code></pre> <h3>输入</h3> <ul> <li><code>low/</code>下的低光RGB帧</li> <li><code>low/</code>下的逐帧事件切片</li> <li>存储为<code>lowlight_event.npz</code>的场景级事件流</li> </ul> <h3>输出 / 监督信息</h3> <ul> <li>存储为<code>normal/depth.npz</code>的对齐深度序列</li> </ul> <h3>重要说明</h3> <ul> <li>数据集按场景/视频数量划分，而非按帧数划分。</li> <li>文件夹名称本身表示划分分配。</li> <li>工作坊版本中的事件切片经过修剪以避免重复计算重叠事件。</li> <li>训练或评估时间模型时，请保持场景级时间顺序不变。</li> </ul>

Submission

<h2>提交格式</h2> <p> 参赛者应为每个测试场景提交一个预测深度序列。 预测深度应与相应场景的帧顺序对齐。 </p> <h3>推荐布局</h3> <pre><code>submission/ &lt;scene_name&gt;/ normal/ depth.npz </code></pre> <h3>深度文件格式</h3> <p> 每个 <code>depth.npz</code> 文件应包含相应场景的预测深度数组。 推荐使用形状为 <code>[T, H, W, 1]</code> 的4D张量以兼容现有DVD风格工具。 评估脚本也可能支持形状为 <code>[T, H, W]</code> 的3D张量。 </p> <h3>提交要求</h3> <ul> <li>提交所有必需测试场景的预测结果。</li> <li>保留原始场景名称。</li> <li>保留每个场景内的时间帧顺序。</li> <li>不要在提交中包含真实标签文件。</li> <li>在上传前将 <code>submission/</code> 文件夹压缩为zip文件。</li> </ul>

Evaluation

<h2>评估</h2> <p> 提交的方案将通过将预测的深度序列与隐藏的地面真实深度标注进行比较来评估。 </p> <h3>指标</h3> <p> 将报告以下指标： </p> <ul> <li>AbsRel / <code>abs_relative_difference</code>（绝对相对差异）</li> <li>RMSE / <code>rmse_linear</code>（线性均方根误差）</li> <li>delta1 / <code>delta1_acc</code></li> <li>delta2 / <code>delta2_acc</code></li> <li>delta3 / <code>delta3_acc</code></li> <li>SILog / <code>silog_rmse</code>（尺度不变对数RMSE）</li> </ul> <h3>排名指标</h3> <p> 最终排名指标将在官方评估阶段之前公布。在启动阶段，建议使用 <strong>AbsRel</strong> 作为主要排名指标，将其余指标作为诊断指标报告。 </p> <h3>评估协议</h3> <ul> <li>预测按场景逐一评估。</li> <li>每个预测的深度序列应与对应的输入序列时间对齐。</li> <li>无效或缺失的预测可能导致评估失败或扣分。</li> </ul>

Rules

<h2>规则和条款</h2> <h3>总则</h3> <ul> <li>参赛者应使用官方提供的训练数据及组织方明确允许的任何额外资源。</li> <li>参赛者不得检查、标注或逆向工程隐藏测试标签。</li> <li>参赛者应在生成预测时保持场景级时间顺序。</li> <li>每支队伍应仅提交遵循规定提交格式的有效预测文件。</li> </ul> <h3>外部数据</h3> <p> 外部数据和预训练模型的使用政策将在官方竞赛启动前公布。 </p> <h3>提交配额</h3> <p> 提交配额：待定。 </p> <h3>时间安排</h3> <ul> <li>开发阶段：待定</li> <li>验证阶段：待定</li> <li>最终测试阶段：待定</li> <li>获胜者公布：待定</li> </ul> <h3>许可证和条款</h3> <ul> <li>数据集许可证：待定</li> <li>代码和基线许可证：待定</li> <li>参赛队伍通过参与表示同意竞赛规则和平台条款。</li> <li>组织方可在提前通知的情况下更新提交格式或评估脚本。</li> </ul>
