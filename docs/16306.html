Overview

## 概述 **PCBA标准到真实场景大挑战**聚焦于**真实制造检验的跨域视觉问答（VQA）**。 印刷电路板组装（PCBA）检验不仅需要视觉识别。实际工厂决策取决于**工业标准**和**真实生产线图像**。可靠的检验模型必须理解制造规则、识别与上下文相关的缺陷、推理可能的原因，并提供可执行的答案。 本挑战评估多模态模型从**标准衍生知识**泛化到**密集、异质真实PCBA图像**的能力。邀请参赛者开发VQA系统，能够解读标准文档、识别组件和缺陷、执行计数任务，并在现实视觉变化下进行推理。 参赛者可在[HuggingFace](https://huggingface.co/datasets/aimmifm/PCBA_Standard-to-Real_Challenge)上通过申请数据集进行注册，并通过官方竞赛平台提交预测文件。 ## 任务 参赛者需回答PCBA相关视觉问题，并为每个测试样本提交预测答案。基准包含四大VQA类别： 1. **基于标准的知识问答** 源自工业标准的问题，涵盖标准解读、缺陷原因分析和处理决策。 2. **事实性** 感知为主的问题，包括组件类型识别、安装面识别、缺陷存在检测和缺陷类型识别。 3. **定量推理** 数字短答题，包括组件计数和管脚/引线计数。 4. **属性推理** 关于视觉属性的问题，如主要组件的形状。 ## 数据集 已发布的数据集 https://huggingface.co/datasets/aimmifm/PCBA_Standard-to-Real_Challenge 由两个互补来源构建： * **标准衍生VQA** 由工业包装标准构建并转换为VQA项目。该子集聚焦于标准知识、缺陷原因分析和决策制定。 * **真实场景VQA** 源自工厂生产线数据。该子集聚焦于真实制造图像和实际视觉变化下的检验推理。 公开分割包含： | 分割 | 问题数量 | | ---- | -------: | | 训练 | 8,200 | | 测试 | 8,200 | | 合计 | 16,400 |

Evaluation

## 评估 所有提交都在官方Codabench服务器上自动评估。参与者必须为发布的公开测试问题提交一个预测文件。公开排行榜报告单一的**总体得分**。 ## 提交格式 对于Codabench提交，参与者必须上传单个`.zip`文件。`.zip`压缩包必须在根级别包含一个名为`submission.csv`的CSV文件： ```text your_submission.zip └── submission.csv ``` `submission.csv`文件必须包含恰好两列： ```csv qid,answer 1,B 2,12 3,E ``` 提交规则： * `qid`必须完全匹配`vqa_test_public.json`中的问题标识符 * `answer`必须仅包含最终预测 * 对于多选题，按照问题中显示的选项键精确提交，例如`A`、`B`、`C`、`D`、`E`或`F` * 对于计数问题，以纯字符串形式提交整数，例如`12` * 文件必须为每个测试问题包含一行 * 缺少行、重复的`qid`值或未知的`qid`值将导致提交无效 * 不要在提交压缩包中将`submission.csv`放置在额外的子文件夹内 公开测试JSON不公开任务类型注释。任务分组由官方评估服务器内部处理。 ## 官方指标 每个测试问题在内部属于以下任务组之一： * `standard_knowledge` * `component_type` * `mount_side` * `defect_existence` * `defect_type` * `count_component` * `count_pin_lead` * `attribute_reasoning` 任务级指标： * **准确率**用于`standard_knowledge`、`component_type`、`mount_side`、`defect_type`和`attribute_reasoning` * **F1得分**用于`defect_existence` * **标准化MAE**用于`count_component`和`count_pin_lead` 对于`defect_existence`，官方得分使用异常阳性类的二元F1。 对于计数任务，官方标准化得分为： ```text count_score_t = max(0, 1 - MAE_t / C_t) ``` 其中固定的标准化常数为： * `C_count_component = 138` * `C_count_pin_lead = 260` ## 总体得分 每个任务级指标首先转换为相同的`0-1`范围。最终公开排行榜得分随后计算为跨任务组的加权平均，使用评估集中的任务比例： ```text Overall Score = 100 * sum_t (w_t * s_t) ``` 其中： * `s_t`是任务`t`的任务级得分 * `w_t = n_t / N` * `n_t`是任务`t`的评估问题数 * `N`是评估问题总数 排行榜在`0-100`范围内显示最终**总体得分**。 ## 评估说明 * 官方Codabench服务器是得分的唯一有效来源 * 组织者方面的任务注释对参与者隐藏 * 多选题的无效选项标签被计为错误答案 * 计数问题的非整数答案被视为无效计数预测 * 组织者可能会在私有评估数据上另外运行相同的评分协议以进行验证或赛后分析
