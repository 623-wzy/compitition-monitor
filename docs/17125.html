# 第2届低级视觉前沿研讨会（LoViF）@ ECCV 2026 # AIGC图像压缩挑战赛 ## 开发阶段 在开发阶段，提交物将使用保真度导向和感知导向的度量在低比特率约束下进行评估。评估度量包括PSNR、MS-SSIM、LPIPS、DISTS和比特率（BPP）。PSNR和MS-SSIM均在RGB域中计算。 本挑战赛要求参赛者在**单一比特率操作点**提交结果。整个数据集的平均比特率约束为： ``` average_bpp ≤ 0.02 ``` 违反比特率约束的提交将被视为无效，不会出现在排行榜上。 开发阶段的排行榜排名由以下加权得分决定： ``` Final_Score = PSNR + 10 × MS_SSIM + 40 × (1 - LPIPS) + 40 × (1 - DISTS) ``` 下面提供了度量计算的简化实现。 ```python import sys import pyiqa def main(rec_path, gt_path): psnr_m = pyiqa.create_metric('psnr') ms_m = pyiqa.create_metric('ms_ssim') lp_m = pyiqa.create_metric('lpips') dists_m = pyiqa.create_metric('dists') psnr = float(psnr_m(rec_path, gt_path).detach().cpu().squeeze().item()) msssim_final = float(ms_m(rec_path, gt_path).detach().cpu().squeeze().item()) lpips_final = float(lp_m(rec_path, gt_path).detach().cpu().squeeze().item()) dists_final = float(dists_m(rec_path, gt_path).detach().cpu().squeeze().item()) final_score = psnr + 10.0 * msssim_final - 20.0 * lpips_final - 25.0 * dists_final + 45.0 if __name__ == '__main__': main(sys.argv[1], sys.argv[2]) ``` ## 测试阶段 开发阶段结束后，打算参加最终测试阶段的团队必须向挑战赛组织者提交解码器可执行文件以及相应的模型检查点。为了控制模型复杂性和参数规模，提交的解码器和模型检查点的总大小不能超过4 GB。**参赛者在最终评估中必须仅使用提交的代码和相应的模型检查点**。 最终排名将由公开度量集合以及组织者的有效性和可重复性验证决定。提交的方法将从重建保真度和感知质量两个角度进行评估，并额外关注AIGC特定的细节，如文本可读性、合成纹理保留和语义一致性。 请参考提交页面以获取详细的提交说明和所需的文件格式。 ## 最终测试阶段 - 初步结果[待公布] 评估完成后，初步结果将在此发布。
