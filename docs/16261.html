Overview

# 基于事件的多模态感知挑战赛：语义分割赛道 @ ECCV 2026 欢迎参加**基于事件的多模态感知挑战赛**的**语义分割赛道**，官方举办地为**ECCV 2026**！ ## 🎯 概述与目标 本挑战赛旨在使用**事件摄像头和帧数据**推进**语义分割**技术发展。与传统RGB摄像头不同，事件摄像头以高时间分辨率、低延迟和高动态范围异步捕捉亮度变化，特别适合应对高速运动、低光环境和强光照变化等具有挑战性的场景。 本赛道的目标是评估和推动稳健的语义分割方法，使其能够有效利用**事件流和强度帧**的互补信息。参与者需要开发能够从多模态输入预测精确的像素级语义标签的模型，重点关注在具有挑战性视觉条件下的泛化性、鲁棒性和场景理解能力。 ## 📝 任务描述 参赛者需要为测试集提交预测的语义分割图。每个预测应遵循官方提交格式，并对应提供的事件和帧输入数据。 该任务重点关注： - **语义分割**：从事件和帧数据预测像素级语义标签。 - **多模态融合**：有效结合异步事件流和图像帧进行语义感知。 - **像素级场景理解**：将每个有效像素分配到预定义的语义类别。 - **鲁棒感知**：处理低照度、高动态范围等具有挑战性的视觉条件。 ## 📅 重要日期 所有截止日期均为**世界协调时间午夜11:59**，除非另有说明。 - **2026年5月13日**：挑战赛网站开放。 - **2026年5月15日**：训练集发布。 - **2026年6月20日**：测试集与评估服务器上线。 - **2026年6月30日**：挑战赛提交截止。 - **2026年7月5日**：挑战赛结果公布。 - **2026年7月10日**：技术报告截止。 ## 📊 数据集 本赛道使用**[CoSEC数据集](https://cosec.org.cn/)**和**[REAL数据集](https://github.com/jinghan-xu/NEC-Diff)**的子集。数据集提供多模态传感器数据，包括图像帧、事件流和语义分割标注。 本赛道发布的数据包括： - **图像帧**：单目摄像头捕捉的RGB帧。 - **事件流**：由事件摄像头记录的异步事件数据。 - **语义分割标签**：为监督学习和验证提供的像素级语义标签真实数据。 训练集用于模型开发和性能验证。测试集用于官方评估，其语义分割标签真实数据保持隐藏。 本赛道的数据集统计信息汇总如下： | 数据集 | 样本数量 | |---|---:| | 训练集 | 2569 | | 测试集 | 待发布 | ## ⚠️ 挑战赛规则 为保持公平性和科学诚实性，所有参与者必须严格遵守挑战赛规则： - **禁止使用测试集训练**：测试集不得用于训练、验证、模型选择或手动调优。 - **禁止测试集标签探测**：参与者不得尝试从评估服务器推断、重建或提取隐藏的语义分割标签真实数据。 - **禁止手动标注**：提交的预测必须由自动化方法生成。禁止使用手动创建、手动修正或手动调优的预测。 - **禁止评估服务器滥用**：严格禁止任何尝试利用评估系统、提交反馈或排行榜行为来获得不公平优势的行为。 - **外部资源披露**：任何外部数据集、预训练模型、合成数据或其他

# 评估指标与提交指南 ## 提交格式 参赛者需要为测试集中的所有样本提交语义分割预测结果。每个预测应该是对应一个测试图像帧的像素级语义标签图。 提交文件必须是单个 `.zip` 压缩包。该压缩包应包含所有预测的语义分割掩码，同时保持所需的序列级目录结构。 每个预测文件必须满足以下要求： - 文件扩展名必须为 `.png`。 - 每个预测文件必须包含语义分割标签图。 - 预测标签图必须是**单通道PNG掩码**。 - 预测标签图必须保存为整数像素值。建议使用 `uint8`。 - 预测标签图的尺寸必须与对应的真值语义分割标签相匹配。 - 文件名必须与对应的测试样本名称完全相同。 - 预测的标签值必须遵循官方语义类别定义。 - 有效的标签值为 `0` 到 `18` 之间的整数。 - 不在预定义类别集合中的无效标签值将被视为错误预测。 ## 语义标签格式 提交的语义分割图应直接存储像素级语义类ID。 每个像素值根据官方类别定义代表其预测的语义类别。例如，像素值 `13` 表示类别 `汽车`，而像素值 `0` 表示类别 `道路`。 预测掩码应存储**原始像素值形式的类ID**，而非可视化颜色。参赛者不应提交彩色RGB可视化结果作为预测掩码。 有效的语义类别如下： | 类别ID | 类别名称 | | ---: | --- | | 0 | 道路 | | 1 | 人行道 | | 2 | 建筑 | | 3 | 墙体 | | 4 | 栅栏 | | 5 | 杆 | | 6 | 交通灯 | | 7 | 交通标志 | | 8 | 植被 | | 9 | 地形 | | 10 | 天空 | | 11 | 行人 | | 12 | 骑行者 | | 13 | 汽车 | | 14 | 卡车 | | 15 | 公交车 | | 16 | 火车 | | 17 | 摩托车 | | 18 | 自行车 | ## 提交结构 下面展示了提交结构的示例： ```text submission.zip ├── Day_Campus_000/ │ └── segment_co/ │ ├── 000000.png │ ├── 000001.png │ ├── 000002.png │ └── ... ├── Day_Campus_001/ │ └── segment_co/ │ ├── 000000.png │ ├── 000001.png │ └── ... └── ... ``` 提供了一份虚拟提交文件，以帮助参赛者验证所需的文件结构和提交格式。 👉 **下载虚拟提交文件** - **百度网盘**: [dummy_submission.zip](https://pan.baidu.com/s/1Q8BdgnOfex4XX3JUFgS-JA?pwd=HUST) - **Google Drive**: [dummy_submission.zip](https://drive.google.com/file/d/1-KTZAXIKoWlOQsBBnyNZHebq_WJ-Juqe/view?usp=drive_link) 参赛者可在准备自己的预测文件时将虚拟提交作为参考。 例如，对于每个测试图像帧： ```text test/Day_Campus_000/img_co_left/000000.png ``` 提交的预测应为： ```text Day_Campus_000/segment_co/000000.png ``` 请确保提交的 `.zip` 文件直接包含序列文件夹。不要包含不必要的父目录。 ## 如何提交 1. **准备预测结果**：为所有测试样本生成语义分割预测。 2. **转换标签格式**：根据官方类ID映射，将模型输出转换为所需的语义标签格式。 3. **保存预测结果**：将每个预测保存为单通道 `.png` 掩码，类ID为 `0` 到 `18` 的整数值。 4. **检查文件结构**：确保目录结构与所需格式匹配。每个测试样本应恰好有一个对应的预测文件。 5. **压缩结果**：将序列文件夹压缩为单个 `.zip` 压缩包。 6. **上传**：导航到竞赛页面的**我的提交**标签页，选择活跃阶段，并上传你的 `.zip` 文件。
