# 通用AIGC音视频检测 本竞赛聚焦于**通用AIGC音视频检测**。为应对高级AI生成视频伪造（如Sora2、Veo3.1）日益增长的威胁，我们基于**MVAD数据集**（发布为DDL-GAV）提出了一项大规模多模态检测挑战赛，包含超过**200,000个真实-伪造音视频样本**。数据集覆盖逼真和动画两种视觉风格、四个内容类别（人类、动物、物体和场景）、三种典型跨模态伪造模式，以及超过**23种AI生成技术**，包括文本生成视频、图像生成视频、视频生成视频和音频生成视频合成。本竞赛旨在推进多模态深度伪造检测的研究与应用，支持**AI向善**的可持续发展。 竞赛链接：https://www.codabench.org/competitions/15769/ 如有任何疑问，请发送邮件至：17318508682@163.com #### 注：测试集已于2026年5月15日在GitHub库中正式发布。 --- ## 评估指标 本挑战赛包含**两个分层检测任务**：二元伪造检测和四类模态分类。 ### 任务定义 - 真-真（RR）：音频和视频均真实。 - 伪-真（FR）：视频伪造，音频真实。 - 真-伪（RF）：视频真实，音频伪造。 - 伪-伪（FF）：音频和视频均伪造。 ### 评估指标 检测性能使用**AUC、ACC、F1和AP**进行评估。 ### 最终评分计算 - 二元分类得分 = 0.2×AUC + 0.3×ACC + 0.3×AP + 0.2×F1 - 四类分类得分 = 0.2×AUC + 0.3×ACC + 0.3×AP + 0.2×F1 - **最终总得分 = 0.4×二元分类得分 + 0.6×四类分类得分** ## 提交格式 参赛者必须提交**两个预测文件**，打包在**单个`.zip`压缩包**中。 ### 打包规则 - zip压缩包可以使用**任意名称**（`submission.zip`、`result.zip`等）。 - 必须恰好包含两个文件，名称**必须**为`binary.txt`和`four_class.txt`（区分大小写——不要重命名）。 - 两个文件可以位于zip根目录或子文件夹中——两种方式都可接受。 - 仅提交预测结果——**不要**提交模型代码。 ### 1. `binary.txt` — 二元分类（真实/伪造） 每行格式：`video_path 0_pred 1_pred` - `0_pred` = **真实**的概率 - `1_pred` = **伪造**的概率 - 可选的标题行`video_path,0_pred,1_pred`被允许且会被跳过。 ``` video_path,0_pred,1_pred 000bfd940cf44f998b1b4803d48d7d05.mp4 0.5066 0.4934 0016150625f846bf8feebc6e5c0831b8.mp4 0.5100 0.4900 001eb668e3ba4ecd89417be16ca7b613.mp4 0.6434 0.3566 ``` ### 2. `four_class.txt` — 四类分类 每行格式：`video_path 0_pred 1_pred 2_pred 3_pred` - 类别索引：`0 = 真-真(RR)`、`1 = 伪-伪(FF)`、`2 = 伪-真(FR)`、`3 = 真-伪(RF)` - 可选的标题行被允许且会被跳过。 ``` video_path,0_pred,1_pred,2_pred,3_pred 000bfd940cf44f998b1b4803d48d7d05.mp4 0.1292 0.0075 0.3836 0.4796 0016150625f846bf8feebc6e5c0831b8.mp4 0.2880 0.0590 0.2999 0.3531 001eb668e3ba4ecd89417be16ca7b613.mp4 0.3144 0.0230 0.3682 0.2944 ``` ### 重要说明 - **`video_path`必须与测试集文件名完全匹配**（发布测试数据中的哈希值`.mp4`文件名）。不匹配的文件名被视为缺失，评分时使用默认均匀概率。 - 每行内，使用**空格或逗号**作为分隔符保持一致——**不要**混合使用`逗号+空格`。 - 概率无需恰好求和为1——评分器会自动对每行进行归一化。 - **测试集中的每个样本应同时出现在两个文件中。**缺失的行将用默认概率填充，会降低您的分数。 ## 奖励与认可 - 优秀方案将被邀请在IJCAI进行展示。 - **前1-5名团队**将获得官方排名证书。
